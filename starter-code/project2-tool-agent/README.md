# 工具调用 Agent

> 智能体工程师培养计划 - Phase 1 项目 2
> 
> 构建一个能主动调用外部工具的智能 Agent，实现从"被动回答"到"主动执行"的质变

## 📋 项目简介

本项目是一个 **工具调用 Agent**，展示了 LLM Agent 的核心能力：根据用户输入，自动选择合适的工具并执行，最终生成回答。

### 核心功能

- ✅ 天气查询（实时 API）
- ✅ 数学计算（支持科学计算）
- ✅ 日历操作（模拟）
- ✅ ReAct 循环（推理-行动-观察）
- ✅ 命令行交互界面
- ✅ 工具调用日志
- ✅ 多轮对话支持

## 🏗️ 技术架构

### 技术栈

- **Agent 框架**: LangGraph / 手写 ReAct 循环
- **LLM**: OpenAI API (gpt-4o-mini)
- **工具**: Python 函数 + OpenAI Function Calling
- **界面**: 命令行

## 🚀 快速开始

### 1. 环境准备

```bash
# 进入项目目录
cd starter-code/project2-tool-agent

# 创建虚拟环境（推荐）
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt
```

### 2. 配置 API Key

```bash
# 复制环境变量示例文件
cp .env.example .env

# 编辑 .env 文件，填入你的 OpenAI API Key
# OPENAI_API_KEY=your_openai_api_key_here
```

### 3. 运行 Agent

```bash
# 运行 Agent（命令行界面）
python src/agent.py
```

## 🔧 核心模块说明

### 1. agent.py

Agent 主文件，实现 ReAct 循环：

- `ToolAgent` 类：封装整个 Agent
- `run()` 方法：执行 ReAct 循环
- 工具注册表：管理所有可用工具
- 对话历史：保持多轮对话上下文

### 2. tools/

工具目录，每个工具都是一个 Python 函数：

- `weather.py`: 天气查询（调用 wttr.in API）
- `calculator.py`: 数学计算（支持基本运算和科学计算）
- `calendar.py`: 日历操作（模拟）

### 3. prompts.py

定义 Agent 的 System Prompt：

- 角色定义
- 工具使用说明
- 行为规则

## ⚠️ 注意事项

1. **API Key 安全**: 不要将 `.env` 文件提交到 Git
2. **网络连接**: 天气查询需要网络连接
3. **工具描述**: 工具的描述要清晰，LLM 靠描述判断何时调用
4. **错误处理**: 工具失败时不要直接抛异常，返回错误信息让 LLM 决策
