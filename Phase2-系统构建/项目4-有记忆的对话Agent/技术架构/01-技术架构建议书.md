## 一、业务诉求

### 1.1 为什么需要"记忆"

老板，**没有记忆的 Agent 是个金鱼**——聊到第 5 轮就忘了你是谁、刚才问了什么。

学员会遇到的真实痛点：

| 场景 | 没记忆 | 有记忆 |
|------|--------|--------|
| 用户："我叫小明，在深圳" | ❌ 3 轮后忘了 | ✅ 永远记得 |
| 用户："刚才那个产品多少钱？" | ❌ 不知道指什么 | ✅ 知道 |
| 用户："我之前让你写过周报" | ❌ 不知道 | ✅ 能调取历史 |
| 第二次使用 | ❌ 从零开始 | ✅ 延续上次对话 |

### 1.2 业务需求清单

#### 功能性需求

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F1 | 记住用户基本信息（姓名、城市、职业） | P0 |
| F2 | 跨会话延续（用户离开后回来还能记得） | P0 |
| F3 | 主动检索相关历史（"上次说的那个项目"） | P1 |
| F4 | 摘要压缩长对话（避免 Context 爆炸） | P1 |
| F5 | 用户画像（用户偏好简短/详细回答） | P2 |

#### 非功能性需求

| 维度 | 指标 |
|------|------|
| 性能 | 单轮响应 < 2s，记忆检索 < 200ms |
| 容量 | 单用户最多存储 1000 条对话 |
| 持久化 | 服务重启不丢失 |
| 隐私 | 用户可随时删除自己的记忆 |

### 2.2 记忆分层模型

**核心设计：三层记忆架构**

| 层级 | 存储 | 用途 | 生命周期 |
|------|------|------|----------|
| **工作记忆** | LLM Context Window | 当前对话 | 单次会话 |
| **短期记忆** | PostgreSQL | 原始对话历史 | N 天（可配） |
| **长期记忆** | 向量数据库 | 摘要 + 事实 + 画像 | 永久（可删除） |

**为什么分三层**：

- **工作记忆**：实时，零延迟，但容量小（128K Token）
- **短期记忆**：原始信息无损，但检索慢
- **长期记忆**：压缩后，检索快，但损失细节

**协作流程**：

```
用户输入
  ↓
① 工作记忆（最近 5 轮） → 直接放 Context
  ↓
② 长期记忆检索（Top-3 相关事实） → 注入 Context
  ↓
③ LLM 推理 → 生成回答
  ↓
④ 写入短期记忆（PostgreSQL）
  ↓
⑤ 异步：定时摘要 → 提取事实 → 写入长期记忆
```

## 三、选型决策

### 3.1 Agent 框架选型

| 方案 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **LangGraph** | 状态管理清晰、可视化、内置 Checkpoint | 学习曲线陡 | ⭐⭐⭐⭐⭐ |
| LangChain Memory | 简单、上手快 | 状态管理弱、难扩展 | ⭐⭐⭐ |
| AutoGen | 多 Agent 强 | 单 Agent 复杂 | ⭐⭐ |

**推荐 LangGraph**——状态机适合做记忆流，**`MemorySaver` 是一等公民**。

### 3.2 短期记忆存储

| 方案 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **PostgreSQL** | 事务、JSON 支持、生态好 | 需要运维 | ⭐⭐⭐⭐⭐ |
| SQLite | 零配置、文件式 | 并发差、不适合生产 | ⭐⭐⭐ |
| Redis | 快、TTL 自然 | 不适合存大量历史 | ⭐⭐ |

**推荐 PostgreSQL**——结构化数据友好，**支持 JSONB 存对话内容**。

### 3.3 长期记忆存储

复用**项目 3 的 Chroma**——存事实摘要 + 用户画像。

### 3.4 摘要模型

**选型**：

- **轻量场景**：gpt-4o-mini（便宜，够用）
- **高质量场景**：gpt-4o（贵 30 倍）

**建议**：

| 场景 | 模型 | 成本 |
|------|------|------|
| 普通用户 | gpt-4o-mini | ~$0.01/次摘要 |

**不要用**开源模型做摘要——质量差 30%，成本节省有限。

## 四、关键技术点

### 4.1 LangGraph 状态机设计

```python
from typing import TypedDict, Annotated
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.postgres import PostgresSaver
from langgraph.graph.message import add_messages
import operator

# 1. 定义状态
class AgentState(TypedDict):
    messages: Annotated[list, add_messages]  # 自动合并
    user_id: str
    long_term_memories: list  # 检索到的事实
    user_profile: dict  # 用户画像

# 2. 构建图
workflow = StateGraph(AgentState)

# 3. 节点
async def recall_memories(state: AgentState):
    """检索长期记忆"""
    query = state["messages"][-1].content
    memories = await retrieve_long_term(query, state["user_id"])
    return {"long_term_memories": memories}

async def generate_response(state: AgentState):
    """生成回答"""
    prompt = build_prompt(
        state["messages"],
        state["long_term_memories"],
        state["user_profile"]
    )
    response = await llm.ainvoke(prompt)
    return {"messages": [response]}

async def save_memory(state: AgentState):
    """保存到长期记忆"""
    await extract_and_save_facts(
        state["messages"],
        state["user_id"]
    )
    return {}

# 4. 边
workflow.add_node("recall", recall_memories)
workflow.add_node("generate", generate_response)
workflow.add_node("save", save_memory)

workflow.add_edge(START, "recall")
workflow.add_edge("recall", "generate")
workflow.add_edge("generate", "save")
workflow.add_edge("save", END)

# 5. Checkpoint（短期记忆）
checkpointer = PostgresSaver.from_conn_string(DB_URL)
app = workflow.compile(checkpointer=checkpointer)
```

### 4.2 长期记忆检索

```python
async def retrieve_long_term(query: str, user_id: str, top_k: int = 3):
    """从向量库检索相关记忆"""
    collection = chroma_client.get_collection("long_term_memory")
    
    # 加用户过滤（重要！不能跨用户检索）
    results = collection.query(
        query_texts=[query],
        n_results=top_k,
        where={"user_id": user_id},  # 强制过滤
        where_document={"type": {"$in": ["personal_info", "preference"]}}
    )
    
    return [
        {
            "content": doc,
            "relevance": 1 - distance
        }
        for doc, distance in zip(results["documents"][0], results["distances"][0])
    ]
```

### 4.3 事实提取（写入长期记忆）

```python
EXTRACT_FACT_PROMPT = """分析以下对话，提取值得记住的事实。

对话:
{messages}

提取规则:
1. 个人信息（姓名、城市、职业、年龄）
2. 偏好（喜欢/不喜欢、习惯）
3. 重要决策或承诺
4. 未来计划

输出 JSON 格式:
[
  {{"type": "personal_info", "content": "用户叫小明"}},
  {{"type": "preference", "content": "用户喜欢简短回答"}}
]

如果没有值得记住的，返回空数组 []
"""

async def extract_and_save_facts(messages, user_id):
    """提取事实并存储"""
    prompt = EXTRACT_FACT_PROMPT.format(messages=format_messages(messages))
    response = await llm.ainvoke(prompt)
    
    try:
        facts = json.loads(response.content)
    except json.JSONDecodeError:
        return
    
    # 存储到向量库
    for fact in facts:
        embedding = await embed_model.aembed(fact["content"])
        collection.add(
            ids=[f"{user_id}_{int(time.time())}_{i}"],
            documents=[fact["content"]],
            embeddings=[embedding],
            metadatas=[{
                "user_id": user_id,
                "type": fact["type"],
                "created_at": datetime.now().isoformat()
            }]
        )
```

### 4.4 摘要压缩

```python
SUMMARIZE_PROMPT = """将以下对话历史压缩成 200 字以内的摘要。

要求:
1. 保留关键信息和决策
2. 保留未完成的待办
3. 保留人物和关系
4. 忽略寒暄和重复

对话:
{history}

摘要:"""

async def summarize_old_conversations(user_id: str):
    """摘要旧的对话（保留最近 5 轮原始，其他的压缩）"""
    history = await db.get_conversations(user_id, limit=20)
    
    if len(history) <= 5:
        return
    
    # 摘要前 N 轮
    old_msgs = history[:-5]
    summary = await llm.ainvoke(
        SUMMARIZE_PROMPT.format(history=format_messages(old_msgs))
    )
    
    # 存储摘要到长期记忆
    await store_summary(user_id, summary.content)
    
    # 删除原始（节省空间）
    await db.delete_conversations(user_id, keep=5)
```

### 4.5 用户画像

```python
async def update_user_profile(user_id: str, signals: dict):
    """根据用户行为更新画像"""
    profile = await get_user_profile(user_id)
    
    # 推理偏好
    if signals.get("avg_response_length") and len(profile.get("messages", [])) > 10:
        # 累计 10 轮后开始判断
        short_count = sum(1 for m in profile["messages"] if len(m) < 50)
        if short_count / len(profile["messages"]) > 0.7:
            profile.setdefault("preferences", {})["answer_style"] = "short"
    
    # 存储
    await db.upsert_user_profile(user_id, profile)
```

-- 1. 对话表（短期记忆原始数据）
CREATE TABLE conversations (
    id BIGSERIAL PRIMARY KEY,
    session_id VARCHAR(64) NOT NULL,
    user_id VARCHAR(64) NOT NULL,
    role VARCHAR(20) NOT NULL,  -- 'user' / 'assistant' / 'system'
    content TEXT NOT NULL,
    metadata JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_conv_user_session ON conversations(user_id, session_id, created_at);

-- 2. 用户画像表
CREATE TABLE user_profiles (
    user_id VARCHAR(64) PRIMARY KEY,
    name VARCHAR(100),
    preferences JSONB DEFAULT '{}',
    facts JSONB DEFAULT '{}',  -- 提取的事实
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 3. 摘要表
CREATE TABLE conversation_summaries (
    id BIGSERIAL PRIMARY KEY,
    user_id VARCHAR(64) NOT NULL,
    session_id VARCHAR(64),
    summary TEXT NOT NULL,
    message_count INT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

## 六、API 设计

```python
from fastapi import FastAPI, Depends
from pydantic import BaseModel

app = FastAPI()

class ChatRequest(BaseModel):
    message: str
    session_id: str
    stream: bool = True

@app.post("/chat")
async def chat(req: ChatRequest, user = Depends(get_current_user)):
    config = {"configurable": {"thread_id": req.session_id, "user_id": user.id}}
    
    if req.stream:
        async def event_generator():
            async for event in graph.astream(
                {"messages": [HumanMessage(content=req.message)]},
                config=config
            ):
                yield f"data: {json.dumps(event)}\n\n"
        return StreamingResponse(event_generator(), media_type="text/event-stream")
    else:
        result = await graph.ainvoke(
            {"messages": [HumanMessage(content=req.message)]},
            config=config
        )
        return {"response": result["messages"][-1].content}

@app.delete("/memory/{user_id}")
async def delete_user_memory(user_id: str):
    """用户删除自己的所有记忆（GDPR 合规）"""
    await db.delete_all(user_id)
    collection.delete(where={"user_id": user_id})
    return {"status": "deleted"}
```
