## 一、业务诉求

### 1.1 为什么需要多 Agent

老板，**单 Agent 有 3 大局限**：

| 局限 | 表现 |
|------|------|
| **任务复杂** | 一份调研报告要"搜索-分析-写作-审校"，单 Agent 容易跑偏 |
| **上下文爆炸** | 多步任务的历史塞满 Context，LLM 抓不住重点 |
| **质量难保证** | 单一模型风格单一，没有"自审"机制 |

**多 Agent 的核心价值**：

> **"分而治之"**——复杂任务拆给多个 Agent，每个 Agent 专注一件事，最后协作完成。

**真实业务场景**（学员会遇到的）：

| 场景 | 多 Agent 协作 |
|------|---------------|
| 调研报告生成 | 搜索员 + 分析师 + 写作者 + 审校员 |
| 自动化营销 | 文案 Agent + 设计 Agent + 数据 Agent + 发布 Agent |
| 客服系统 | 接待 Agent + 技术支持 Agent + 售后 Agent |
| 代码开发 | 产品 Agent + 开发 Agent + 测试 Agent + Review Agent |

### 1.2 业务需求清单

#### 功能性需求

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F1 | 至少 3 个 Agent 协作完成报告 | P0 |
| F2 | Agent 间通信机制 | P0 |
| F3 | 任务编排（顺序/并行/条件） | P0 |
| F4 | 人工审核节点（Human-in-the-loop） | P1 |
| F5 | 任务可观测（每步在做什么） | P1 |
| F6 | 重试和恢复 | P2 |

#### 非功能性需求

| 维度 | 指标 |
|------|------|
| 协作 | 至少支持 4 Agent 串行 |
| 可靠性 | 单步失败不导致整体崩溃 |
| 可观测 | 每步的输入输出可追溯 |
| 性能 | 4 Agent 串行完成 < 5 分钟 |

## 二、整体架构

### 2.1 架构图（报告生成为例）

```
用户输入主题: "AI 在医疗领域的应用"
   ↓
┌────────────────────────────────────────────┐
│  Orchestrator (编排器)                     │
│  决定任务顺序、并行、依赖                   │
└────────────────────────────────────────────┘
   ↓
   ① 研究员 Agent (Researcher)
   ↓
   - 搜索资料 (Tavily API)
   - 整理信息 (10 条要点)
   ↓
   ② 分析师 Agent (Analyst)
   ↓
   - 提取关键洞察
   - 数据可视化建议
   ↓
   ③ 写作者 Agent (Writer)  ← ④ 审校员 (Reviewer) 并行
   ↓                          ↓
   - 撰写报告 (3000 字)      - 检查事实错误
   - 引用格式                - 检查逻辑漏洞
                              - 评分 1-10
   ↓
   ⑤ 主编 Agent (Editor)
   ↓
   - 根据审校意见修改
   - 输出最终版本
   ↓
   返回给用户
```

### 2.2 通信模式

**3 种核心模式**：

| 模式 | 场景 | 优点 | 缺点 |
|------|------|------|------|
| **串行**（Sequential） | 有明确依赖（A→B→C） | 简单、清晰 | 慢 |
| **并行**（Parallel） | 互相独立 | 快 | 资源占用高 |
| **混合**（Hierarchical） | 复杂任务 | 灵活 | 复杂度高 |

**本项目采用混合模式**：

```
研究员 → 分析师 → [写作者 ∥ 审校员] → 主编 → 输出
              串行            并行
```

## 三、选型决策

### 3.1 多 Agent 框架对比

| 框架 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **LangGraph** | 灵活、可视化、状态管理强 | 学习曲线陡 | ⭐⭐⭐⭐⭐ |
| **AutoGen** (微软) | 对话驱动、自然 | 状态管理弱、难调试 | ⭐⭐⭐ |
| **CrewAI** | 易上手、角色清晰 | 灵活性低 | ⭐⭐⭐⭐ |
| **AgentScope** | 国内开源、中文好 | 生态小 | ⭐⭐⭐ |

**推荐 LangGraph**——理由：

1. **状态机模型** 适合复杂协作（**项目 4 已铺垫**）
2. **可视化**（LangGraph Studio）
3. **可恢复**（Checkpoint + 时间旅行调试）
4. **灵活**——任何编排都能写

**次选 CrewAI**——如果学员嫌 LangGraph 复杂，可以切。

### 3.2 任务队列（可选）

**生产环境建议加任务队列**：

| 方案 | 适用 |
|------|------|
| **Celery** | Python 生态成熟、Redis/RabbitMQ 后端 |
| **RQ** | 轻量、Redis 后端、Python 友好 |
| **Temporal** | 工业级、状态管理最强、学习曲线陡 |
| **直接 LangGraph** | 简单场景、单机够用 |

**基础版**：直接用 LangGraph（无需队列）
**生产版**：Celery + Redis（支持分布式）

### 3.3 通信协议

**2 种模式**：

| 模式 | 描述 | 适用 |
|------|------|------|
| **共享状态** | 所有 Agent 操作同一个 State | LangGraph 默认 |
| **消息传递** | Agent 间显式发消息 | AutoGen 默认 |

**本项目用共享状态**——**更简单，更易调试**。

## 四、关键技术点

### 4.1 Agent 角色设计

**每个 Agent 4 大要素**：

```python
class Agent:
    role: str          # "研究员"
    goal: str          # "收集关于 X 的最新研究"
    backstory: str     # "你是一个有 10 年经验的市场调研员..."
    tools: list        # [search_tool, fetch_tool]
```

**角色 Prompt 模板**：

```python
RESEARCHER_PROMPT = """你是市场研究员，专注于 {topic}。

你的目标:
1. 搜索最新（2025-2026）的资料
2. 整理 10 个关键要点
3. 每个要点附上来源链接

工作方式:
- 使用 search_web 工具搜索
- 使用 fetch_url 工具获取详细内容
- 不要自己编造数据

输出格式 (JSON):
{
    "topic": "...",
    "key_points": [
        {"point": "...", "source": "...", "year": 2025}
    ]
}
"""
```

### 4.2 LangGraph 完整实现

```python
from typing import TypedDict, Annotated
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
import operator

# 1. 定义共享状态
class ReportState(TypedDict):
    # 用户输入
    topic: str
    user_id: str
    
    # 流程数据
    research_data: dict | None      # 研究员输出
    analysis: dict | None           # 分析师输出
    draft: str | None               # 写作者输出
    review: dict | None             # 审校员输出
    final_report: str | None        # 主编输出
    
    # 元数据
    messages: Annotated[list, add_messages]
    iteration: int                  # 迭代次数
    quality_score: int              # 质量评分

# 2. 定义 Agent 节点
async def researcher_node(state: ReportState):
    """研究员"""
    prompt = RESEARCHER_PROMPT.format(topic=state["topic"])
    response = await llm.ainvoke(prompt)
    data = json.loads(extract_json(response.content))
    return {"research_data": data, "messages": [response]}

async def analyst_node(state: ReportState):
    """分析师"""
    prompt = ANALYST_PROMPT.format(
        topic=state["topic"],
        research=json.dumps(state["research_data"], ensure_ascii=False)
    )
    response = await llm.ainvoke(prompt)
    return {"analysis": json.loads(extract_json(response.content))}

async def writer_node(state: ReportState):
    """写作者"""
    prompt = WRITER_PROMPT.format(
        topic=state["topic"],
        research=state["research_data"],
        analysis=state["analysis"]
    )
    response = await llm.ainvoke(prompt)
    return {"draft": response.content}

async def reviewer_node(state: ReportState):
    """审校员"""
    prompt = REVIEWER_PROMPT.format(
        topic=state["topic"],
        draft=state["draft"]
    )
    response = await llm.ainvoke(prompt)
    review = json.loads(extract_json(response.content))
    return {
        "review": review,
        "quality_score": review.get("score", 0)
    }

async def editor_node(state: ReportState):
    """主编（根据审校意见修改）"""
    prompt = EDITOR_PROMPT.format(
        draft=state["draft"],
        review=state["review"]
    )
    response = await llm.ainvoke(prompt)
    return {"final_report": response.content}

# 3. 条件分支
def should_rewrite(state: ReportState):
    """判断是否需要重写"""
    if state["quality_score"] < 7:
        return "writer"  # 回到写作者
    return "editor"

# 4. 构建图
workflow = StateGraph(ReportState)

workflow.add_node("researcher", researcher_node)
workflow.add_node("analyst", analyst_node)
workflow.add_node("writer", writer_node)
workflow.add_node("reviewer", reviewer_node)
workflow.add_node("editor", editor_node)

# 串行
workflow.add_edge(START, "researcher")
workflow.add_edge("researcher", "analyst")
workflow.add_edge("analyst", "writer")
workflow.add_edge("writer", "reviewer")

# 条件分支
workflow.add_conditional_edges(
    "reviewer",
    should_rewrite,
    {
        "writer": "writer",     # 不合格重写
        "editor": "editor"      # 合格送审
    }
)
workflow.add_edge("editor", END)

# 5. 编译
app = workflow.compile()

# 6. 调用
result = await app.ainvoke({
    "topic": "AI 在医疗领域的应用",
    "user_id": "user_123"
})
print(result["final_report"])
```

### 4.3 Human-in-the-Loop（人工审核节点）

```python
from langgraph.checkpoint.memory import MemorySaver
from langgraph.types import interrupt, Command

# 1. 在审校后插入人工审核
async def human_review_node(state: ReportState):
    """暂停，等待人工审核"""
    # 触发中断，前端显示给用户
    user_input = interrupt({
        "type": "human_review",
        "draft": state["draft"],
        "ai_review": state["review"]
    })
    
    # 用户决策：approve / reject
    if user_input["action"] == "approve":
        return {"quality_score": 10}  # 强制通过
    else:
        return {
            "review": {
                "feedback": user_input["feedback"],
                "score": 5
            },
            "quality_score": 5
        }

# 2. 前端处理中断
# 当 graph.stream 暂停时，前端收到 interrupt 信号
# 用户操作后，调用 graph.invoke(Command(resume=user_input))
```

### 4.4 失败重试

```python
async def safe_node(node_fn):
    """节点装饰器：失败重试"""
    async def wrapper(state):
        last_error = None
        for attempt in range(3):
            try:
                return await node_fn(state)
            except Exception as e:
                last_error = e
                await asyncio.sleep(2 ** attempt)  # 指数退避
                logger.warning(f"重试 {attempt+1}/3: {e}")
        
        # 3 次都失败
        return {
            "error": str(last_error),
            "should_continue": False
        }
    return wrapper

# 使用
@safe_node
async def researcher_node(state):
    ...
```

### 4.5 可观测性

```python
import time

async def timed_node(node_fn):
    """节点装饰器：计时 + 日志"""
    async def wrapper(state):
        start = time.time()
        node_name = node_fn.__name__
        
        logger.info(f"[{node_name}] 开始")
        result = await node_fn(state)
        elapsed = time.time() - start
        
        logger.info(f"[{node_name}] 完成, 耗时 {elapsed:.2f}s")
        
        # 存储到 state
        if "node_timings" not in state:
            state["node_timings"] = {}
        state["node_timings"][node_name] = elapsed
        
        return result
    return wrapper
```
