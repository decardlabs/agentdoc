## 一、业务诉求

### 1.1 为什么 Agent 需要执行代码

老板，**LLM 自己不会算数、不会画图、不会处理文件**——它只会"说"。

学员会遇到的问题：

| 场景 | 纯 LLM | 代码执行 Agent |
|------|--------|----------------|
| "计算 1234 × 5678 + 9999" | ❌ 算错（LLM 数学能力差） | ✅ 让 Python 算 |
| "分析这个 CSV 的销售趋势" | ❌ 编造数据 | ✅ 写 pandas 代码跑 |
| "画一个折线图" | ❌ 只能描述 | ✅ matplotlib 出图 |
| "把这段 JSON 转成 Excel" | ❌ 不能操作文件 | ✅ 写转换脚本 |

**这是 LLM 的核心能力扩展**——从"动嘴"升级到"动手"。

### 1.2 业务需求清单

#### 功能性需求

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F1 | 自动生成 Python 代码 | P0 |
| F2 | 在沙箱中安全执行 | P0 |
| F3 | 错误捕获与自动修复 | P0 |
| F4 | 处理文件（CSV、Excel、图片） | P1 |
| F5 | 安装第三方包 | P1 |
| F6 | 超时和资源限制 | P0 |

#### 非功能性需求

| 维度 | 指标 |
|------|------|
| 安全性 | 用户代码**绝对不能**影响主服务 |
| 隔离性 | 不同用户的代码互不干扰 |
| 性能 | 单次执行 < 30s |
| 资源 | 单次执行 CPU < 2 核，内存 < 2GB |

## 三、选型决策

### 3.1 沙箱方案对比

老板，**这是整个项目最关键的选型**。沙箱选错了，**安全出事故就完蛋**。

| 方案 | 隔离级别 | 易用性 | 性能 | 成本 | 推荐度 |
|------|----------|--------|------|------|--------|
| **E2B (Firecracker)** | **VM 级** | ⭐⭐⭐⭐⭐ | 快 | $0.05/h | ⭐⭐⭐⭐⭐ |
| Docker | 容器级 | ⭐⭐⭐⭐ | 中 | 免费 | ⭐⭐⭐⭐ |
| Pyodide (浏览器) | 进程内 | ⭐⭐ | 慢 | 免费 | ⭐⭐ |
| RestrictedPython | 进程内 | ⭐⭐ | 快 | 免费 | ⭐ |
| 直接 exec() | 无 | ⭐⭐⭐⭐⭐ | 快 | 免费 | **❌ 绝对禁止** |

**推荐 E2B**——理由：

1. **真正的 VM 隔离**（Firecracker microVM），比 Docker 安全 10 倍
2. **即开即用**——2 行代码启动沙箱
3. **支持任意 Python 包**——`pip install` 就行
4. **成本可控**——$0.05/h，1 个沙箱跑 1 小时只要 3 毛钱
5. **可观测**——内置日志、监控

**Docker 备选**——如果成本敏感：

- 适合内部工具、低风险场景
- 需要自己写沙箱管理代码
- **必须**用 `--read-only --network=none --memory=2g` 限制

### 3.2 LLM 选型

**代码生成能力排名**（2026 年 6 月）：

| 模型 | 代码能力 | 速度 | 成本 | 推荐场景 |
|------|----------|------|------|----------|
| Claude Sonnet 4.5 | ⭐⭐⭐⭐⭐ | 中 | 中 | **复杂任务首选** |

**推荐**：

### 3.3 自修复机制

**方案对比**：

| 方案 | 原理 | 准确率 | 成本 |
|------|------|--------|------|
| **简单重试** | 报错就把错误信息回传 LLM 重新生成 | 70% | 低 |
| **链式反思** | 多次重试 + 让 LLM 解释为什么错 | 85% | 中 |
| **测试驱动** | 写代码 + 写测试，测试失败就重试 | 90% | 高 |
| **专家系统** | 错误分类（语法/逻辑/依赖），针对性修复 | 95% | 高 |

**推荐**：
- **基础版**：简单重试（70% 准确率够用）
- **挑战版**：链式反思 + 错误分类

## 四、关键技术点

### 4.1 E2B 沙箱集成

**E2B 是什么**：

> E2B = "Executor for AI"，专门为 AI Agent 设计的云端代码沙箱。

**最小化集成代码**（5 行启动）：

```python
from e2b_code_interpreter import Sandbox

# 1. 创建沙箱（默认 1C2G）
with Sandbox() as sandbox:
    # 2. 执行代码
    result = sandbox.run_code("print('Hello, World!')")
    print(result.text)  # Hello, World!
    
    # 3. 获取图表
    for chart in result.charts:
        chart.show()  # matplotlib 图表
        # 或保存
        with open("chart.png", "wb") as f:
            f.write(chart.png)
```

**生产级封装**：

```python
class CodeSandbox:
    """E2B 沙箱封装"""
    
    def __init__(self, timeout: int = 30):
        self.timeout = timeout
    
    async def execute(self, code: str, files: dict = None):
        """执行代码，可选上传文件"""
        async with Sandbox() as sandbox:
            # 1. 上传文件
            if files:
                for filename, content in files.items():
                    sandbox.filesystem.write(filename, content)
            
            # 2. 执行
            try:
                result = await asyncio.wait_for(
                    sandbox.run_code_async(code),
                    timeout=self.timeout
                )
                
                # 3. 处理结果
                return {
                    "success": True,
                    "stdout": result.text,
                    "error": None,
                    "charts": [
                        {
                            "type": chart.type,
                            "png": base64.b64encode(chart.png).decode() if chart.png else None
                        }
                        for chart in result.charts
                    ]
                }
            except asyncio.TimeoutError:
                return {
                    "success": False,
                    "error": f"执行超时（>{self.timeout}秒）",
                    "stdout": ""
                }
            except Exception as e:
                return {
                    "success": False,
                    "error": str(e),
                    "stdout": ""
                }
```

### 4.2 代码生成 Prompt

```python
CODE_GEN_PROMPT = """你是一个 Python 数据分析专家。

用户问题: {user_query}

可用文件:
{file_list}

要求:
1. 写完整的、可执行的 Python 代码
2. 不要省略（...）或伪代码
3. 处理常见的边缘情况（如空数据、类型错误）
4. 如果需要可视化，使用 matplotlib，保存为 png
5. 在代码最后用 print 输出关键结果

输出格式:
```python
# 你的代码
```

只输出代码，不要其他解释。
"""
```

**关键点**：
- **完整的**（不省略）——LLM 经常偷懒写 `...`
- **自包含**——所有 import 都在代码里
- **打印结果**——让 LLM 后面能解读

### 4.3 自修复循环

```python
class SelfHealingCodeAgent:
    def __init__(self, max_retries=3):
        self.max_retries = max_retries
        self.sandbox = CodeSandbox()
    
    async def run(self, user_query: str, files: dict = None):
        """执行 + 自修复"""
        
        code = None
        history = []
        
        for attempt in range(self.max_retries):
            # 1. 生成代码（第一次基于用户问题，后续基于错误）
            if attempt == 0:
                code = await self.generate_code(user_query, files)
            else:
                code = await self.fix_code(code, history[-1]["error"], user_query)
            
            # 2. 执行
            result = await self.sandbox.execute(code, files)
            history.append({
                "attempt": attempt + 1,
                "code": code,
                "result": result
            })
            
            # 3. 成功就退出
            if result["success"]:
                break
            
            # 4. 检查错误是否可修复
            if not self.is_fixable(result["error"]):
                break  # 不可修复的错误（如包不存在）
        
        # 5. 解读结果
        final_answer = await self.interpret(user_query, history)
        return {
            "answer": final_answer,
            "code": history[-1]["code"],
            "executions": history,
            "attempts": len(history)
        }
    
    async def generate_code(self, query, files):
        """生成代码"""
        prompt = CODE_GEN_PROMPT.format(
            user_query=query,
            file_list=list(files.keys()) if files else "无"
        )
        response = await llm.ainvoke(prompt)
        return self.extract_code(response.content)
    
    async def fix_code(self, code, error, original_query):
        """基于错误修复代码"""
        prompt = f"""原任务: {original_query}

原代码:
```python
{code}
```

错误信息:
{error}

请修复代码。输出完整的修复后代码。
```python
# 修复后的代码
```
"""
        response = await llm.ainvoke(prompt)
        return self.extract_code(response.content)
    
    def is_fixable(self, error):
        """判断错误是否可修复"""
        unfixable = [
            "ModuleNotFoundError",  # 依赖问题
            "FileNotFoundError",    # 文件不存在
            "MemoryError",          # 内存不足
        ]
        return not any(e in error for e in unfixable)
    
    def extract_code(self, text):
        """从 LLM 响应中提取代码块"""
        match = re.search(r'```python\n(.*?)\n```', text, re.DOTALL)
        return match.group(1) if match else text
```

#### 防线 5：AST 静态检查

```python
import ast

FORBIDDEN_NODES = {
    ast.Call: ["os.system", "subprocess.Popen", "eval", "exec"],
}

def static_check(code: str):
    """执行前静态扫描危险代码"""
    tree = ast.parse(code)
    
    for node in ast.walk(tree):
        if isinstance(node, ast.Call):
            func_name = ast.unparse(node.func) if hasattr(ast, 'unparse') else ''
            if any(f in func_name for f in ["os.system", "subprocess", "eval", "exec"]):
                raise SecurityError(f"危险调用: {func_name}")
```

## 五、API 设计

```python
from fastapi import FastAPI, UploadFile
from pydantic import BaseModel

app = FastAPI()

class CodeRequest(BaseModel):
    query: str
    code: str | None = None  # 可选，用户已写好代码

@app.post("/analyze")
async def analyze(
    query: str,
    files: list[UploadFile] = None
):
    """分析文件（自动生成代码 + 执行）"""
    
    # 1. 上传文件到沙箱
    file_dict = {}
    if files:
        for f in files:
            content = await f.read()
            file_dict[f.filename] = content
    
    # 2. 执行
    agent = SelfHealingCodeAgent()
    result = await agent.run(query, file_dict)
    
    return result

@app.post("/execute")
async def execute(req: CodeRequest):
    """直接执行用户提供的代码"""
    agent = SelfHealingCodeAgent()
    result = await agent.run(req.query, code=req.code)
    return result
```
