## 一、开场：用"翻车现场"震撼学员

老板，这个项目**必须先让学员看到 LLM 的无能**。

演示流程：

> "现在我让 GPT-4o 算一个数：1234 × 5678 + 9999 等于多少？"

```python
# 让 LLM 算
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "1234 × 5678 + 9999 = ?"}]
)
print(response.choices[0].message.content)
# 输出："1234 × 5678 + 9999 = 7,007,891"  ← 实际是 7,008,991
```

**让学员亲眼看到**——LLM 数学能力差 30%。这叫 **"LLM 不会算数"** 现象。

> "这就是为什么需要代码执行。**LLM 不擅长精确计算，但擅长'写计算代码'**。让 Python 算 1+1 比让 GPT 算 1+1 准 100 倍。"

**强调安全**（必讲）：

> "你说'让 AI 写代码 + 执行'，**第一反应应该是害怕**。这意味着用户代码能在你的服务器上跑——**搞不好就是 RCE 漏洞**。
> 
> 所以这个项目的核心不是'写代码'，是**'安全地写代码'**。"

## 二、3 个核心概念，必须讲透

### 概念 1：沙箱 = Agent 的"武器库"

**学员最大误区**：

```python
# ❌ 学员的第一反应
user_code = input("Enter code: ")
exec(user_code)  # RCE 漏洞，100% 被攻击
```

**讲师必讲**：

> "为什么 `exec()` 是灾难？因为用户可以写 `exec('import os; os.system("rm -rf /")')`——**你亲手把核弹按钮交给黑客**。"

**沙箱分类**（按隔离级别）：

```
进程内 (RestrictedPython)  ← 隔离最弱，只能挡 30% 攻击
   ↓
容器 (Docker)  ← 隔离中等，挡 80%
   ↓
VM (E2B / Firecracker)  ← 隔离最强，挡 99.9%
   ↓
物理机 (AWS Nitro)  ← 隔离最强，成本最高
```

**讲师金句**：

> "**生产环境永远不要 `exec()`**。**沙箱是必须的**，但沙箱也有 bug——**多层防御才是正解**。"

### 概念 2：自修复 = Agent 的"试错能力"

**学员误区**：以为"生成代码 → 执行" 就完事了。

**真相**：

> LLM 写的代码**有 30-50% 概率第一次跑不通**。SyntaxError、NameError、TypeError 全都可能。

**自修复的 3 个层级**：

| 层级 | 机制 | 修复率 |
|------|------|--------|
| L1 | 报错就把错误信息回传，让 LLM 重新生成 | 70% |
| L2 | 加反思："你为什么错了？下次注意什么？" | 85% |
| L3 | 错误分类 + 针对性修复 | 95% |

**演示**（必做）：

```
用户: "画一个 (0,0) 到 (10,20) 的折线图"

LLM 生成:
plt.plot([0,10], [0,20])
plt.show()  ← 没保存

执行 → 成功（前端能看到）但用户没拿到图

自修复: 错误信息 = "用户没收到图表"
LLM 重新生成: plt.savefig('chart.png')
执行 → 成功 + 返回文件
```

**讲师金句**：

> "**Agent 的智能 = 第一次就做对 + 错了能改**。**自修复是 Agent 区别于一次性脚本的核心**。"

### 概念 3：5 道防线（生产标配）

**讲师画图**：

```
用户代码
   ↓
[1] AST 静态检查 ← 拦截明显危险代码
   ↓
[2] E2B 沙箱 ← VM 隔离
   ↓
[3] 超时控制 ← 30 秒硬超时
   ↓
[4] 包白名单 ← 只允许装指定包
   ↓
[5] 网络隔离 ← 关闭外网
   ↓
执行
```

**5 道防线**：

| 防线 | 拦什么 | 拦不住什么 |
|------|--------|------------|
| AST 静态检查 | `os.system` / `subprocess` | 运行时才能发现的逻辑 |
| E2B 沙箱 | 任何能想到的攻击 | 沙箱本身的 0day |
| 超时 | 死循环 / 慢代码 | 30 秒内完成的恶意行为 |
| 包白名单 | 装恶意包 | 合法包的恶意用法 |
| 网络隔离 | 攻击内网 | 暂存数据外泄 |

**讲师金句**：

> "**没有 100% 安全的沙箱**。**多层防御 = 攻破一道还有 4 道**。"

## 三、其他概念，根据学员水平选讲

### 概念 4：测试驱动

**进阶内容**（挑战版）：

```python
# 不只是写代码，还自动写测试
code_prompt = """
写代码实现 X。
同时写 3 个测试用例验证代码正确性。
"""

# 执行 → 测试失败 → 重新生成 → 测试通过
```

**为什么有效**：

- 测试 = Agent 自己验证答案
- 90% 的简单错误，测试能直接抓到
- **让 Agent 像程序员一样工作**

### 概念 5：流式输出

**学员的体验问题**：

```
执行 30 秒 → 用户看到 30 秒空白 → "死机了？"
```

**解决**：

```python
async def stream_execution(code):
    async with Sandbox() as sandbox:
        # 启动执行（不等待完成）
        process = await sandbox.process.start(f"python -c '{code}'")
        
        # 流式返回 stdout
        async for line in process.stdout:
            yield f"data: {line}\n\n"
```

**讲师金句**：

> "**30 秒的等待 = 80% 用户流失**。**流式输出 = 让用户觉得'在干活'**。"

### Demo 4：自修复（30 分钟）

```python
async def self_healing_agent(user_query: str, max_retries=3):
    history = []
    
    for attempt in range(max_retries):
        # 1. 生成代码（首次基于问题，后续基于错误）
        if attempt == 0:
            code = await generate_code(user_query)
        else:
            code = await fix_code(
                history[-1]["code"],
                history[-1]["error"],
                user_query
            )
        
        # 2. 执行
        with Sandbox() as sandbox:
            try:
                result = sandbox.run_code(code, timeout=30)
                if result.error:
                    history.append({
                        "code": code,
                        "error": result.error
                    })
                else:
                    return {
                        "success": True,
                        "code": code,
                        "attempts": attempt + 1,
                        "output": result.text
                    }
            except Exception as e:
                history.append({"code": code, "error": str(e)})
    
    return {
        "success": False,
        "attempts": max_retries,
        "history": history
    }
```

**演示**：
- 故意制造错误：`code = "print(undefined_var)"` 
- 看 Agent 怎么修复

### Demo 5：安全防护（30 分钟）

```python
import ast

class SecurityChecker:
    FORBIDDEN = ["os.system", "subprocess", "eval", "exec"]
    
    def check(self, code: str):
        """静态检查"""
        try:
            tree = ast.parse(code)
        except SyntaxError:
            return False, "代码语法错误"
        
        for node in ast.walk(tree):
            if isinstance(node, ast.Call):
                func_name = self.get_func_name(node)
                if any(f in func_name for f in self.FORBIDDEN):
                    return False, f"禁止调用: {func_name}"
        
        return True, None
    
    def get_func_name(self, node):
        """获取函数名"""
        if isinstance(node.func, ast.Name):
            return node.func.id
        elif isinstance(node.func, ast.Attribute):
            return f"{self.get_func_name(node.func.value)}.{node.func.attr}"
        return ""
```

**演示**：
- `exec('print(1)')` → 拦截 ✅
- `__import__('os').system('whoami')` → 拦截 ✅
- `import os; os.listdir('.')` → **不拦截**（合法但有风险）

**讲师要点**：

> "AST 检查能拦 80% 明显攻击，**剩下的 20% 靠沙箱兜底**。**没有银弹**。"

## 六、必讲的 6 个踩坑点

### 坑 1：`exec()` 是 RCE 漏洞

**学员第一反应**：

```python
# ❌ 千万别这么写
user_code = input("Code: ")
exec(user_code)
```

**讲师必讲**：

> "代码里出现 `exec(user_input)` 直接打回重做——**这是入门必踩的雷**。"

### 坑 2：包安装超时

**现象**：`pip install pandas` 跑 5 分钟还没装完。

**对策**：
- 预装基础包（pandas、numpy、matplotlib）
- 白名单常用包
- **冷启动慢的包用 lazy load**

### 坑 3：图表返回失败

**现象**：执行成功但前端没收到图。

**原因**：
- `plt.show()` 不保存
- E2B 用 `result.charts` 而不是 `plt.savefig()`

**对策**：
```python
# ✅ 正确方式（E2B 自动捕获）
plt.plot(...)
plt.title(...)  # 必须有 title
# 不要 plt.show()
```

### 坑 4：自修复死循环

**现象**：修了 5 次还是同样的错。

**对策**：
- 设置 `max_retries=3`
- 检查错误是否可修复（`is_fixable`）
- 不可修复就**优雅失败**，告诉用户

### 坑 5：文件路径错误

**现象**：
```python
df = pd.read_csv("data.csv")  # FileNotFoundError
```

**原因**：文件没传到沙箱。

**对策**：
```python
# 上传时打印日志
print(f"上传文件: {filename}, 大小: {len(content)} bytes")
# 沙箱里再 listdir 确认
print(os.listdir("."))
```

### 坑 6：超时设置太小

**现象**：分析大文件 30 秒不够。

**对策**：
- 简单任务 30s
- 复杂任务 60-120s
- **永远设超时**（不设就是炸弹）

## 七、Q&A 预案

**Q1：E2B 免费吗？**
A：有免费额度（$100/月试用），生产环境按使用付费（$0.05/h）。**1 个沙箱跑 1 小时只要 3 毛钱**。

**Q2：能不能用 Docker 代替 E2B？**
A：能，但需要自己写沙箱管理代码。Docker 隔离级别比 E2B 弱，**生产环境建议 E2B**。

**Q3：自修复能 100% 成功吗？**
A：不能。**70-95% 取决于任务复杂度**。简单任务 90%+，复杂任务 60%。

**Q4：用户能传任何 Python 包吗？**
A：不能。**生产环境必须白名单**。可选：①预装 90% 常用包 ②按需安装（白名单内） ③写明申请流程。

**Q5：E2B 安全吗？会不会被黑客攻破？**
A：E2B 基于 Firecracker microVM，**和 AWS Lambda 同款隔离**。比 Docker 安全 10 倍。但**没有 100% 安全**，**多层防御是必须的**。

**Q6：执行结果怎么传给前端？**
A：文本直接返回，图表用 base64 编码。**生产环境建议存 OSS/S3，前端 CDN 加载**。

**Q7：能支持 R 吗？Julia 吗？**
A：能。E2B 默认支持 Python，但可以装其他语言。**最常用还是 Python**。

**Q8：怎么防止 Agent 死循环烧钱？**
A：①E2B 沙箱自动超时 ②单任务 Token 上限 ③每日总成本告警。
