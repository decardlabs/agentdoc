## 2. 业务诉求与技术挑战

### 2.1 业务诉求

项目 6 你已经把多 Agent 系统跑起来了。但现在的状态是：

```
你的系统：
  - 在你 Mac 上能跑
  - 在同事的 Windows 上跑不了（Python 版本不对）
  - 换台电脑要重新装依赖、重新配置
  - 线上部署？手动 scp 文件 + ssh 手动启动，一个人干运维
```

**容器化要解决的核心问题**：

| 问题 | 容器化之前 | 容器化之后 |
|------|-----------|-----------|
| 环境不一致 | "我电脑上能跑啊" | 镜像打包，到处一致 |
| 部署流程 | 手动 scp + 手动启动 | 一条命令拉起全套 |
| 回滚能力 | 手动备份 + 手动恢复 | 镜像版本化，秒级回滚 |
| 依赖管理 | requirements.txt 可能冲突 | 镜像锁死完整环境 |
| 团队协作 | "你帮我装一下环境" | docker compose up，完事 |
| 扩容能力 | 手动开新机器 | docker compose up --scale agent=3 |

### 2.2 学员痛点

| 痛点 | 表现 | 根因 |
|------|------|------|
| **不会写 Dockerfile** | 不知道 FROM、RUN、COPY 怎么组合 | 没理解"层"的概念 |
| **镜像太大** | 构建出来 2GB，拉取要 10 分钟 | 基础镜像选错、没清理缓存 |
| **服务间不通** | Agent 连不上 Redis | 不理解 Docker 网络 |
| **数据重启就丢** | docker compose down 后 Redis 数据全没 | 没配 Volume |
| **CI/CD 搭不起来** | GitHub Actions 报一堆错 | Secrets、权限、镜像名拼错 |
| **部署后端口不通** | 云服务器上 curl 能通，外网访问不了 | 安全组没开端口 |

> **架构设计的核心矛盾**：学员想"快速上线"，但容器化、编排、CI/CD、反向代理、HTTPS 每一项都需要理解原理，否则上线就是上线一个定时炸弹。

## 4. 技术选型

### 4.1 核心选型决策表

| 层级 | 候选方案 | 选哪个 | 理由 |
|------|----------|--------|------|
| **容器运行时** | Docker / Podman / Containerd | **Docker** | 生态最成熟、文档最多、社区最大 |
| **编排工具** | Docker Compose / Kubernetes / Nomad | **Docker Compose** | 学习曲线低，单机/少数机器完全够用 |
| **基础镜像** | python:3.11 / python:3.11-slim / alpine | **python:3.11-slim** | ~150MB vs 完整版 ~900MB；alpine 用 musl libc 有兼容问题 |
| **Web 服务器** | Nginx / Caddy / Traefik | **Nginx** | 配置灵活、性能稳定、资源占用低 |
| **SSL 证书** | Let's Encrypt / Cloudflare | **Let's Encrypt** | 免费、自动续期、行业标配 |
| **CI/CD** | GitHub Actions / GitLab CI / Jenkins | **GitHub Actions** | 免费额度够用、YAML 配置简单 |
| **镜像仓库** | Docker Hub / 阿里云 ACR / GHCR | **Docker Hub + 阿里云 ACR** | Docker Hub 国际通用；阿里云 ACR 国内拉取快 |
| **日志方案** | stdout+Docker JSON / ELK / Loki | **Docker JSON Driver** | 轻量、查询方便，配合 max-size 防磁盘写满 |
| **监控** | Prometheus+Grafana / Datadog | **Prometheus+Grafana** | 开源免费、生态成熟 |

### 4.2 Docker vs Podman 对比

| 维度 | Docker | Podman |
|------|--------|--------|
| 架构 | Client-Server（Daemon） | 无 Daemon（fork-exec） |
| root 权限 | 需 Daemon 以 root 运行 | 可 rootless 运行 |
| Compose 支持 | 原生 docker compose | podman-compose 兼容但不完全 |
| 生态/文档 | 最好 | 较少 |
| 安全性 | Daemon 权限大 | 更安全 |
| **推荐场景** | **学习和中小项目** | 安全要求高的企业环境 |

> 选 Docker。遇到问题能搜到答案，这是最重要的考量。

## 5. 关键技术点设计

### 5.1 Dockerfile 多阶段构建

**完整的 Dockerfile**（项目 6 多 Agent 系统专用）：

```dockerfile
# ============================================================
# 阶段 1: 构建依赖 (builder)
# 只安装编译需要的工具，不进入最终镜像
# ============================================================
FROM python:3.11-slim AS builder

WORKDIR /build

# 安装编译依赖（gcc 用于编译 C 扩展包如 cryptography、uvloop）
RUN apt-get update && \
    apt-get install -y --no-install-recommends \
        gcc g++ libffi-dev libpq-dev \
    && rm -rf /var/lib/apt/lists/*

# 先只复制 requirements.txt，利用 Docker 层缓存
COPY requirements.txt .

# 安装到 /install 目录（不污染系统 Python）
RUN pip install --prefix=/install --no-cache-dir \
    --extra-index-url https://download.pytorch.org/whl/cpu \
    -r requirements.txt

# ============================================================
# 阶段 2: 运行时镜像 (runtime)
# 只包含运行需要的最小文件
# ============================================================
FROM python:3.11-slim AS runtime

# 安装运行时系统依赖
RUN apt-get update && \
    apt-get install -y --no-install-recommends curl libpq5 tini \
    && rm -rf /var/lib/apt/lists/* && apt-get clean

WORKDIR /app

# 从 builder 阶段复制已编译的 Python 包
COPY --from=builder /install /usr/local

# 创建非 root 用户（安全实践）
RUN groupadd -r appuser && useradd -r -g appuser -d /app -s /sbin/nologin appuser

# 复制应用代码
COPY --chown=appuser:appuser app/ ./app/
COPY --chown=appuser:appuser pyproject.toml ./

# 设置时区 + Python 优化
ENV TZ=Asia/Shanghai \
    LANG=C.UTF-8 \
    PYTHONUNBUFFERED=1 \
    PYTHONDONTWRITEBYTECODE=1 \
    PATH=/usr/local/bin:$PATH

# 创建日志目录
RUN mkdir -p /app/logs && chown appuser:appuser /app/logs

# 切换到非 root 用户
USER appuser

# 健康检查：每 30 秒检查一次，超时 10 秒，启动后 40 秒开始
HEALTHCHECK --interval=30s --timeout=10s --start-period=40s --retries=3 \
    CMD curl -f http://localhost:8000/health || exit 1

# 暴露端口
EXPOSE 8000

# 使用 tinit 作为 PID 1（正确处理信号，避免僵尸进程）
ENTRYPOINT ["tini", "--"]

# 启动命令
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000", "--workers", "2"]
```

### 5.2 Docker Compose 多服务编排

**完整 docker-compose.yml**（Agent + Redis + Postgres + Chroma + Nginx + Prometheus + Grafana）：

```yaml
services:
  # ============ 主应用 ============
  agent:
    build: { context: ., dockerfile: Dockerfile }
    image: multi-agent:v1.0
    container_name: multi-agent-app
    restart: unless-stopped
    ports: ["${APP_PORT:-8000}:8000"]
    env_file: [.env]
    environment:
      - REDIS_URL=redis://redis:6379/0
      - DATABASE_URL=postgresql://agent:agent123@postgres:5432/agentdb
      - CHROMA_HOST=chroma
      - CHROMA_PORT=8000
      - LOG_LEVEL=${LOG_LEVEL:-INFO}
    depends_on:
      redis: { condition: service_healthy }
      postgres: { condition: service_healthy }
      chroma: { condition: service_started }
    volumes: [./logs:/app/logs]
    networks: [agent-network]
    deploy:
      resources:
        limits: { cpus: '2.0', memory: 2G }
        reservations: { cpus: '0.5', memory: 512M }
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s
    logging:
      driver: json-file
      options: { max-size: "50m", max-file: "5" }

  # ============ Redis ============
  redis:
    image: redis:7-alpine
    container_name: multi-agent-redis
    restart: unless-stopped
    ports: ["${REDIS_PORT:-6379}:6379"]
    volumes: [redis-data:/data]
    command: >
      redis-server --appendonly yes --maxmemory 256mb
      --maxmemory-policy allkeys-lru
      --requirepass ${REDIS_PASSWORD:-redis123}
    networks: [agent-network]
    healthcheck:
      test: ["CMD", "redis-cli", "-a", "${REDIS_PASSWORD:-redis123}", "ping"]
      interval: 10s
      timeout: 5s
      retries: 3
    logging:
      driver: json-file
      options: { max-size: "20m", max-file: "3" }

  # ============ Postgres ============
  postgres:
    image: postgres:16-alpine
    container_name: multi-agent-postgres
    restart: unless-stopped
    ports: ["${POSTGRES_PORT:-5432}:5432"]
    environment:
      POSTGRES_DB: agentdb
      POSTGRES_USER: agent
      POSTGRES_PASSWORD: agent123
      PGDATA: /var/lib/postgresql/data/pgdata
    volumes: [postgres-data:/var/lib/postgresql/data]
    networks: [agent-network]
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U agent -d agentdb"]
      interval: 10s
      timeout: 5s
      retries: 3
    logging:
      driver: json-file
      options: { max-size: "30m", max-file: "3" }

  # ============ Chroma ============
  chroma:
    image: chromadb/chroma:0.4.22
    container_name: multi-agent-chroma
    restart: unless-stopped
    ports: ["${CHROMA_PORT:-8001}:8000"]
    volumes: [chroma-data:/chroma/chroma]
    environment:
      - ANONYMIZED_TELEMETRY=OFF
      - ALLOW_RESET=True
      - IS_PERSISTENT=TRUE
    networks: [agent-network]
    deploy:
      resources:
        limits: { memory: 1G }
    logging:
      driver: json-file
      options: { max-size: "30m", max-file: "3" }

  # ============ Nginx ============
  nginx:
    image: nginx:1.25-alpine
    container_name: multi-agent-nginx
    restart: unless-stopped
    ports: ["80:80", "443:443"]
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./certbot/conf:/etc/letsencrypt:ro
      - ./certbot/www:/var/www/certbot:ro
    depends_on:
      agent: { condition: service_healthy }
    networks: [agent-network]
    logging:
      driver: json-file
      options: { max-size: "20m", max-file: "3" }

  # ============ Prometheus ============
  prometheus:
    image: prom/prometheus:v2.48.0
    container_name: multi-agent-prometheus
    restart: unless-stopped
    ports: ["9090:9090"]
    volumes:
      - ./monitoring/prometheus.yml:/etc/prometheus/prometheus.yml:ro
      - prometheus-data:/prometheus
    command: ['--config.file=/etc/prometheus/prometheus.yml', '--storage.tsdb.retention.time=7d']
    networks: [agent-network]

  # ============ Grafana ============
  grafana:
    image: grafana/grafana:10.3.0
    container_name: multi-agent-grafana
    restart: unless-stopped
    ports: ["3000:3000"]

volumes:
  redis-data:
  postgres-data:
```

### 5.3 GitHub Actions CI/CD 流水线

**完整配置**（.github/workflows/ci-cd.yml）：

```yaml
# ============================================================
# CI/CD Pipeline - 测试、构建、部署一体化
# ============================================================
name: CI/CD Pipeline

on:
  push:
    branches: [main, develop]
    tags: ['v*']
  pull_request:
    branches: [main]

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

env:
  DOCKER_IMAGE: ${{ secrets.DOCKER_USERNAME }}/multi-agent
  ALIYUN_REGISTRY: registry.cn-hangzhou.aliyuncs.com

jobs:
  # ============ Job 1: 测试 ============
  test:
    name: 单元测试 + 代码检查
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
          cache: 'pip'

      - name: 安装依赖
        run: |
          pip install -r requirements.txt
          pip install pytest pytest-cov pytest-asyncio
          pip install flake8 black isort

      - name: 代码格式检查
        run: |
          flake8 app/ --max-line-length=120 --statistics
          black --check app/
          isort --check-only app/

      - name: 运行测试
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
          DATABASE_URL: sqlite:///./test.db
        run: |
          pytest tests/ \
            --cov=app/ --cov-report=term-missing --cov-report=xml \
            --cov-fail-under=70 -v

      - uses: codecov/codecov-action@v3
        with:
          file: ./coverage.xml
        continue-on-error: true

  # ============ Job 2: 构建并推送镜像 ============
  build-and-push:
    name: 构建并推送镜像
    needs: test
    runs-on: ubuntu-latest
    if: github.event_name == 'push'
    permissions:
      contents: read
      packages: write
    steps:
      - uses: actions/checkout@v4

      - uses: docker/setup-buildx-action@v3

      - name: 登录 Docker Hub
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKER_USERNAME }}
          password: ${{ secrets.DOCKER_PASSWORD }}

      - name: 登录阿里云 ACR
        uses: docker/login-action@v3
        with:
          registry: ${{ env.ALIYUN_REGISTRY }}
          username: ${{ secrets.ALIYUN_USERNAME }}
          password: ${{ secrets.ALIYUN_PASSWORD }}
        continue-on-error: true

      - name: 提取标签
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ${{ env.DOCKER_IMAGE }}
          tags: |
            type=semver,pattern={{version}}
            type=sha,prefix=
            type=raw,value=latest,enable={{is_default_branch}}

      - name: 构建并推送
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          platforms: linux/amd64,linux/arm64
          tags: ${{ steps.meta.outputs.tags }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

      - name: 镜像安全扫描 (Trivy)
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: ${{ env.DOCKER_IMAGE }}:latest
          format: 'table'
          exit-code: '0'
          ignore-unfixed: true
          severity: 'CRITICAL,HIGH'

  # ============ Job 3: 部署到服务器 ============
  deploy:
    name: 部署到生产服务器
    needs: build-and-push
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    environment: production
    steps:
      - name: SSH 部署
        uses: appleboy/ssh-action@v1.0.3
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          key: ${{ secrets.SSH_PRIVATE_KEY }}
          script: |
            set -e
            cd /opt/multi-agent
            docker compose pull
            docker compose up -d --remove-orphans
            sleep 10
            docker compose ps
            if docker compose ps | grep -q "unhealthy"; then
              echo "!!! 服务不健康 !!!"
              exit 1
            fi
            docker image prune -f
            docker system prune -f

      - name: 冒烟测试
        run: |
          sleep 15
          HTTP_CODE=$(curl -s -o /dev/null -w "%{http_code}" \
            "http://${{ secrets.SERVER_HOST }}/health" || echo "000")
          if [ "$HTTP_CODE" != "200" ]; then
            echo "健康检查失败 (HTTP $HTTP_CODE)"
            exit 1
          fi
```

### 5.4 Nginx 反向代理 + HTTPS

**完整 Nginx 配置**（nginx/nginx.conf）：

```nginx
# ============================================================
# Nginx 反向代理 - HTTPS 终止 + 限流 + 缓存
# ============================================================

worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
    multi_accept on;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    log_format main '$remote_addr [$time_local] "$request" '
                    '$status $body_bytes_sent '
                    'upstream_response_time=$upstream_response_time '
                    'request_time=$request_time';
    access_log /var/log/nginx/access.log main;

    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;

    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_comp_level 4;
    gzip_types text/plain text/css application/json
               application/javascript text/xml;

    # 限流
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=10r/s;
    limit_req_zone $binary_remote_addr zone=health_limit:10m rate=30r/s;

    # 上游
    upstream agent_backend {
        server agent:8000;
        keepalive 32;
    }

    # HTTP -> HTTPS 重定向
    server {
        listen 80;
        server_name api.yourdomain.com;

        location /.well-known/acme-challenge/ {
            root /var/www/certbot;
        }
        location / {
            return 301 https://$host$request_uri;
        }
    }

    # HTTPS 主配置
    server {
        listen 443 ssl http2;
        server_name api.yourdomain.com;

        # Let's Encrypt 证书
        ssl_certificate /etc/letsencrypt/live/api.yourdomain.com/fullchain.pem;
        ssl_certificate_key /etc/letsencrypt/live/api.yourdomain.com/privkey.pem;
        ssl_protocols TLSv1.2 TLSv1.3;
        ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
        ssl_prefer_server_ciphers off;
        ssl_session_cache shared:SSL:10m;

        # 安全头
        add_header Strict-Transport-Security "max-age=63072000" always;
        add_header X-Content-Type-Options nosniff always;
        add_header X-Frame-Options DENY always;

        # 健康检查（不限流，不计日志）
        location /health {
            limit_req zone=health_limit burst=50 nodelay;
            proxy_pass http://agent_backend/health;
            access_log off;
        }

        # API 请求
        location /api/ {
            limit_req zone=api_limit burst=20 nodelay;
            proxy_pass http://agent_backend;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_set_header Connection "";
            proxy_connect_timeout 60s;
            proxy_send_timeout 120s;
            proxy_read_timeout 300s;
        }

        # 静态资源缓存
        location ~* \.(jpg|jpeg|png|gif|ico|css|js|svg|woff2)$ {
            proxy_pass http://agent_backend;
            expires 30d;
            add_header Cache-Control "public, immutable";
        }

        # 默认
        location / {
            proxy_pass http://agent_backend;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}
```

```python
import structlog
import logging

def setup_logging(log_level: str = "INFO"):
    structlog.configure(
        processors=[
            structlog.contextvars.merge_contextvars,
            structlog.processors.add_log_level,
            structlog.processors.TimeStamper(fmt="iso"),
            structlog.processors.StackInfoRenderer(),
            structlog.processors.format_exc_info,
            structlog.processors.JSONRenderer(),
        ],
        wrapper_class=structlog.make_filtering_bound_logger(
            getattr(logging, log_level.upper())
        ),
        context_class=dict,
        logger_factory=structlog.PrintLoggerFactory(),
        cache_logger_on_first_use=True,
    )

# 使用
logger = structlog.get_logger()
logger.info("agent.task.completed",
           task_id="task_001", duration_ms=1250, tokens_used=3500)
```
