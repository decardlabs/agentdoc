## 2. 业务诉求与技术挑战

### 2.1 业务诉求

项目 7 已经把 Agent 部署上去了。但**生产环境不是跑通就完事**——你得知道：

| 问题 | 真实场景 | 后果 |
|------|----------|------|
| **成本失控** | 某用户疯狂调用 GPT-4o，一天烧了 $200 | 月底账单爆炸，老板找你谈话 |
| **性能恶化** | P95 延迟从 2s 涨到 15s，用户骂娘 | 用户流失 |
| **错误飙升** | 错误率从 0.5% 涨到 8%，没人知道 | 工具 API 挂了，Agent 全在空转 |
| **定位困难** | 用户说"回答不对"，你找不到哪步出了问题 | 无法修复 |
| **幻觉检测** | Agent 偶尔编造信息，但不知道何时发生 | 信任崩塌 |

> 一句话：**没监控的系统 = 闭眼开车**。你不知道速度、不知道油量、不知道前面是不是悬崖。

### 2.2 核心挑战

| 挑战 | 表现 | 影响 |
|------|------|------|
| **Agent 调用链长** | 一次用户请求 = 3-5 次 LLM 调用 + 2-3 次工具调用 | 链路追踪复杂 |
| **Token 成本波动大** | 同一个问题，不同模型成本差 10-100 倍 | 预算不可控 |
| **多模型混用** | gpt-4o-mini / gpt-4o / claude-3.5-sonnet 各有各的定价 | 成本计算复杂 |
| **告警疲劳** | 一天收到 500 条告警，全是低优先级 | 真正重要的告警被淹没 |
| **存储爆炸** | 每次调用存完整 request/response，数据量指数增长 | 存储成本 > LLM 成本 |
| **监控自身拖性能** | 同步写日志，高频调用时变瓶颈 | 监控反成负担 |

> 架构设计的核心矛盾：**监控要全，但不能拖垮系统；告警要快，但不能轰炸人类**。

### 3.2 数据流时序

**单次 LLM 调用的监控流程**：

```
用户请求
  → Agent 接收请求，生成 trace_id
  → 调用 LLM，记录:
      - 开始时间 (start_time)
      - 模型名 (model)
      - 输入 Token (prompt_tokens)
      - 输出 Token (completion_tokens)
      - 延迟 (latency_ms)
      - 状态 (success/error)
  → 计算成本 (cost_usd)
  → 写入 PostgreSQL (异步)
  → 推送 Prometheus Metrics
  → 检查预算阈值
  → 超阈值 → AlertManager → 钉钉/飞书
  → 返回响应
```

### 3.3 四条监控线的职责

| 监控线 | 职责 | 存储 | 查询方式 |
|--------|------|------|----------|
| **Tracing** | 请求级追踪：一次用户请求的完整调用链 | LangSmith / PostgreSQL | LangSmith UI / SQL |
| **Metrics** | 聚合指标：QPS、P95 延迟、错误率、Token/s | Prometheus | PromQL |
| **Logging** | 离散事件：每次调用的输入/输出/错误 | PostgreSQL / 文件 | SQL / grep |
| **Cost** | 成本追踪：按用户/模型/时间的费用 | PostgreSQL | SQL / Streamlit |

## 4. 选型决策

### 4.1 Tracing 平台选型

| 维度 | LangSmith | Phoenix (Arize) | 自建 (OTel + Jaeger) |
|------|-----------|-----------------|----------------------|
| **接入成本** | 3 行环境变量 | pip install + 5 行代码 | 需部署 Jaeger + Collector |
| **LangChain 集成** | 原生，零代码 | 支持，需配置 | 需手写 OTel Span |
| **非 LangChain** | 需手写 wrapper | 支持 OpenTelemetry | 原生支持 |
| **成本可见** | 内置 | 内置 | 需自建 |
| **免费额度** | 5K traces/月 | 本地部署免费 | 完全免费 |
| **数据安全** | 数据在 LangChain 服务器 | 本地部署，数据不出域 | 完全自有 |
| **推荐场景** | LangChain 生态项目 | 需要本地部署/数据敏感 | 大规模/定制化需求 |

**决策**：

- 基础版：**LangSmith**（3 分钟接入，立刻能看到数据）
- 挑战版：**Phoenix**（本地部署，数据不出域）
- 生产版：**OpenTelemetry + Jaeger**（标准化，不绑供应商）

### 4.2 Metrics 平台选型

| 维度 | Prometheus + Grafana | Datadog | New Relic |
|------|---------------------|---------|-----------|
| **成本** | 免费（自建） | $23/host/月起 | $49/user/月起 |
| **部署** | Docker 一键 | SaaS | SaaS |
| **查询语言** | PromQL | 自有 | NRQL |
| **告警** | AlertManager | 内置 | 内置 |
| **自定义指标** | 完全自由 | 有限 | 有限 |
| **学习曲线** | 中等 | 低 | 低 |
| **适用规模** | 中小团队 | 中大团队 | 大团队 |

**决策**：**Prometheus + Grafana**。理由：免费、灵活、生态大、K8s 标配。中小团队没必要上 Datadog。

### 4.3 成本 Dashboard 选型

| 维度 | 自建 SQLite + Streamlit | 自建 PostgreSQL + Grafana | SaaS (LangSmith 内置) |
|------|------------------------|--------------------------|----------------------|
| **开发成本** | 1 天 | 2-3 天 | 0（内置） |
| **灵活性** | 完全自定义 | 自定义 | 有限 |
| **实时性** | 分钟级 | 秒级 | 分钟级 |
| **查询能力** | SQL（受限） | SQL + PromQL | 有限 |
| **扩展性** | 单机 | 可扩展 | 绑定平台 |

**决策**：**PostgreSQL + Streamlit**（基础版）/ **PostgreSQL + Grafana**（生产版）。自建意味着数据在你手里，想怎么查怎么查。

```python
        ]
    )
    import json
    return json.loads(response.choices[0].message.content)

@traceable(run_type="tool", name="call_tool")
def call_tool(intent: dict) -> str:
    # 执行具体工具...
    return "工具执行结果"

@traceable(run_type="llm", name="generate_answer")
def generate_answer(tool_result: str, user_input: str) -> str:
```

### 5.2 OpenTelemetry 标准化追踪

**为什么需要 OpenTelemetry**：LangSmith 是好用的，但它绑定了一家供应商。OTel 是行业标准——换了追踪平台，代码不用改。

```python
# pip install opentelemetry-api opentelemetry-sdk opentelemetry-exporter-otlp

from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor
from opentelemetry.exporter.otlp.proto.grpc.trace_exporter import OTLPSpanExporter

# 1. 初始化 Tracer
provider = TracerProvider()
processor = BatchSpanProcessor(OTLPSpanExporter(endpoint="http://localhost:4317"))
provider.add_span_processor(processor)
trace.set_tracer_provider(provider)

tracer = trace.get_tracer("agent-system")

# 2. 在代码中创建 Span
def agent_run_with_otel(user_input: str):
    with tracer.start_as_current_span("agent_run") as main_span:
        main_span.set_attribute("user_input", user_input)
        main_span.set_attribute("user_id", "user_001")

        # 意图识别
        with tracer.start_as_current_span("identify_intent") as span:
            intent = identify_intent(user_input)
            span.set_attribute("intent", str(intent))
            span.set_attribute("model", "gpt-4o-mini")
            span.set_attribute("prompt_tokens", 150)

        # 工具调用
        with tracer.start_as_current_span("tool_call") as span:
            result = call_tool(intent)
            span.set_attribute("tool_name", intent.get("tool"))
            span.set_attribute("tool_status", "success")

        # 生成回答
        with tracer.start_as_current_span("generate_answer") as span:
            answer = generate_answer(result, user_input)
            span.set_attribute("completion_tokens", 280)

        main_span.set_attribute("total_cost_usd", 0.003)
        return answer
```

### 5.3 Prometheus Metrics 采集

**三种指标类型**：

| 类型 | 用途 | 例子 |
|------|------|------|
| **Counter** | 只增不减的计数器 | 调用总数、Token 总消耗 |
| **Gauge** | 可增可减的当前值 | 当前在线用户、队列长度 |
| **Histogram** | 分布统计 | 延迟分布（P50/P95/P99） |

**指标定义**：

```python
# pip install prometheus-client

from prometheus_client import Counter, Gauge, Histogram, start_http_server

# ============ Counter ============
llm_call_total = Counter(
    "llm_call_total",
    "Total LLM API calls",
    ["model", "status", "user_id"]  # 标签维度
)

token_consumed_total = Counter(
    "token_consumed_total",
    "Total tokens consumed",
    ["model", "token_type"]  # token_type: input / output
)

tool_call_total = Counter(
    "tool_call_total",
    "Total tool calls",
    ["tool_name", "status"]
)

# ============ Gauge ============
active_sessions = Gauge(
    "active_sessions",
    "Currently active user sessions"
)

queue_size = Gauge(
    "request_queue_size",
    "Pending requests in queue"
)

# ============ Histogram ============
llm_call_duration_seconds = Histogram(
    "llm_call_duration_seconds",
    "LLM call latency in seconds",
    ["model"],
    buckets=[0.5, 1.0, 2.0, 5.0, 10.0, 30.0, 60.0]  # 自定义桶
)

agent_run_duration_seconds = Histogram(
    "agent_run_duration_seconds",
    "Full agent run latency in seconds",
    buckets=[1.0, 3.0, 5.0, 10.0, 30.0, 60.0, 120.0]
)

# 启动 Metrics 服务（端口 8000）
start_http_server(8000)
```

**在业务代码中埋点**：

```python
import time

class MonitoredLLMClient:
    """带 Prometheus 监控的 LLM 客户端"""

    def chat(self, messages, model="gpt-4o-mini", user_id="anonymous", **kwargs):
        start_time = time.time()

        try:
            from openai import OpenAI
            client = OpenAI()
            response = client.chat.completions.create(
                model=model, messages=messages, **kwargs
            )

            # Counter +1
            llm_call_total.labels(
                model=model, status="success", user_id=user_id
            ).inc()

            # Token Counter
            prompt_tokens = response.usage.prompt_tokens
            completion_tokens = response.usage.completion_tokens
            token_consumed_total.labels(model=model, token_type="input").inc(prompt_tokens)
            token_consumed_total.labels(model=model, token_type="output").inc(completion_tokens)

            # Histogram 记录延迟
            duration = time.time() - start_time
            llm_call_duration_seconds.labels(model=model).observe(duration)

            return response.choices[0].message.content

        except Exception as e:
            llm_call_total.labels(
                model=model, status="error", user_id=user_id
            ).inc()
            raise
```

### 5.5 Token 成本计算

**多模型差异化定价表**：

| 模型 | 输入价格 ($/1M tokens) | 输出价格 ($/1M tokens) | 上下文窗口 |
|------|----------------------|----------------------|-----------|
| gpt-4o-mini | 0.15 | 0.60 | 128K |
| gpt-4o | 2.50 | 10.00 | 128K |
| gpt-4o (cached) | 1.25 | 10.00 | 128K |
| claude-3.5-sonnet | 3.00 | 15.00 | 200K |
| claude-3-haiku | 0.25 | 1.25 | 200K |
| deepseek-chat | 0.14 | 0.28 | 128K |

**成本计算引擎**：

```python
from dataclasses import dataclass
from typing import Optional
from datetime import date

# 定价表（$/1M tokens）
PRICING_TABLE = {
    "gpt-4o-mini":      {"input": 0.15,  "output": 0.60},
    "gpt-4o":           {"input": 2.50,  "output": 10.00},
    "gpt-4o-cached":    {"input": 1.25,  "output": 10.00},
    "claude-3.5-sonnet": {"input": 3.00,  "output": 15.00},
    "claude-3-haiku":   {"input": 0.25,  "output": 1.25},
    "deepseek-chat":    {"input": 0.14,  "output": 0.28},
}


@dataclass
class CostRecord:
    """单次调用成本记录"""
    call_id: str
    timestamp: str
    user_id: str
    session_id: str
    model: str
    prompt_tokens: int
    completion_tokens: int
    cost_usd: float
    latency_ms: int
    status: str
    error_message: Optional[str] = None


def calculate_cost(model: str, prompt_tokens: int, completion_tokens: int) -> float:
    """计算单次调用成本（美元）

    Args:
        model: 模型名
        prompt_tokens: 输入 Token 数
        completion_tokens: 输出 Token 数

    Returns:
        成本（美元），保留 6 位小数

    Raises:
        ValueError: 模型不在定价表中
    """
    if model not in PRICING_TABLE:
        raise ValueError(f"模型 {model} 不在定价表中，请先添加")

    pricing = PRICING_TABLE[model]
    cost = (prompt_tokens / 1_000_000) * pricing["input"] + \
           (completion_tokens / 1_000_000) * pricing["output"]
    return round(cost, 6)


def calculate_session_cost(records: list[CostRecord]) -> dict:
    """计算一次会话的总成本"""
    total_cost = sum(r.cost_usd for r in records)
    total_tokens = sum(r.prompt_tokens + r.completion_tokens for r in records)

    by_model = {}
    for r in records:
        if r.model not in by_model:
            by_model[r.model] = {"cost": 0.0, "tokens": 0, "calls": 0}
        by_model[r.model]["cost"] += r.cost_usd
        by_model[r.model]["tokens"] += r.prompt_tokens + r.completion_tokens
        by_model[r.model]["calls"] += 1

    return {
        "total_cost_usd": round(total_cost, 4),
        "total_tokens": total_tokens,
        "total_calls": len(records),
        "by_model": by_model,
    }


def calculate_daily_budget(user_id: str, daily_limit: float = 5.0) -> dict:
    """计算用户当日预算使用情况"""
    today = date.today().isoformat()

    # 从数据库查询当日该用户的总成本
    import sqlite3
    with sqlite3.connect("llm_logs.db") as conn:
        result = conn.execute("""
            SELECT COALESCE(SUM(cost_usd), 0)
            FROM llm_calls
            WHERE user_id = ? AND DATE(timestamp) = ?
        """, (user_id, today)).fetchone()

    spent = result[0]
    remaining = daily_limit - spent
    usage_pct = (spent / daily_limit * 100) if daily_limit > 0 else 0

    return {
        "user_id": user_id,
        "date": today,
        "budget_usd": daily_limit,
        "spent_usd": round(spent, 4),
        "remaining_usd": round(remaining, 4),
        "usage_pct": round(usage_pct, 1),
        "over_budget": spent > daily_limit,
    }


# 使用示例
cost = calculate_cost("gpt-4o", prompt_tokens=2000, completion_tokens=500)
print(f"单次成本: ${cost:.4f}")  # $0.0100

budget = calculate_daily_budget("user_001", daily_limit=5.0)
print(f"当日已用: ${budget['spent_usd']:.4f} / ${budget['budget_usd']:.2f} ({budget['usage_pct']:.1f}%)")
```

### 5.8 异常检测

**P95 + 同比环比检测**：

```python
import numpy as np
from datetime import datetime, timedelta
from typing import Optional

class AnomalyDetector:
    """异常检测器——基于统计方法的简单但有效方案"""

    def __init__(self, db_path: str = "llm_logs.db"):
        self.db_path = db_path

    def detect_latency_anomaly(self, window_hours: int = 24,
                                threshold_sigma: float = 3.0) -> list[dict]:
        """检测延迟异常（3-sigma 规则）

        思路：
        1. 取过去 7 天同时段的延迟数据作为基线
        2. 计算均值和标准差
        3. 当前值超过 mean + 3*std 视为异常

        Args:
            window_hours: 检测窗口（小时）
            threshold_sigma: 异常阈值（几倍标准差）
        """
        import sqlite3
        import pandas as pd

        with sqlite3.connect(self.db_path) as conn:
            # 取过去 7 天的数据作为基线
            df = pd.read_sql("""
                SELECT
                    DATE(timestamp) as date,
                    strftime('%H', timestamp) as hour,
                    AVG(latency_ms) as avg_latency,
                    COUNT(*) as call_count
                FROM llm_calls
                WHERE timestamp >= datetime('now', '-7 days')
                AND status = 'success'
                GROUP BY date, hour
            """, conn)

        if len(df) < 24:
            return []

        mean = df['avg_latency'].mean()
        std = df['avg_latency'].std()

        if std == 0:
            return []

        df['z_score'] = np.abs((df['avg_latency'] - mean) / std)
        anomalies = df[df['z_score'] > threshold_sigma]

        return anomalies[['date', 'hour', 'avg_latency', 'z_score']].to_dict('records')

    def detect_cost_anomaly(self, compare_days: int = 7) -> list[dict]:
        """成本环比异常检测

        思路：今天的成本 vs 过去 7 天日均成本
        增长超过 200% 视为异常
        """
        import sqlite3

        with sqlite3.connect(self.db_path) as conn:
            # 今日成本
            today_result = conn.execute("""
                SELECT COALESCE(SUM(cost_usd), 0)
                FROM llm_calls WHERE DATE(timestamp) = DATE('now')
            """).fetchone()
            today_cost = today_result[0]

            # 过去 7 天日均成本
            avg_result = conn.execute("""
                SELECT COALESCE(AVG(daily_cost), 0) FROM (
                    SELECT SUM(cost_usd) as daily_cost
                    FROM llm_calls
                    WHERE DATE(timestamp) >= DATE('now', '-7 days')
                    AND DATE(timestamp) < DATE('now')
                    GROUP BY DATE(timestamp)
                )
            """).fetchone()
            avg_daily_cost = avg_result[0]

        if avg_daily_cost == 0:
            return []

        growth_rate = today_cost / avg_daily_cost
        anomalies = []

        if growth_rate > 3.0:
            anomalies.append({
                "type": "cost_spike",
                "severity": "critical",
                "today_cost": round(today_cost, 2),
                "avg_daily_cost": round(avg_daily_cost, 2),
                "growth_rate": round(growth_rate, 1),
                "message": f"今日成本 ${today_cost:.2f} 是日均 ${avg_daily_cost:.2f} 的 {growth_rate:.1f} 倍"
            })
        elif growth_rate > 2.0:
            anomalies.append({
                "type": "cost_increase",
                "severity": "warning",
                "today_cost": round(today_cost, 2),
                "avg_daily_cost": round(avg_daily_cost, 2),
                "growth_rate": round(growth_rate, 1),
                "message": f"今日成本 ${today_cost:.2f} 是日均 ${avg_daily_cost:.2f} 的 {growth_rate:.1f} 倍"
            })

        return anomalies

    def detect_error_rate_anomaly(self) -> list[dict]:
        """错误率同比异常检测

        当前 1 小时错误率 vs 过去 24 小时平均错误率
        """
        import sqlite3

        with sqlite3.connect(self.db_path) as conn:
            # 当前 1 小时错误率
            current = conn.execute("""
                SELECT
                    SUM(CASE WHEN status = 'error' THEN 1 ELSE 0 END) * 1.0 / COUNT(*) as error_rate
                FROM llm_calls
                WHERE timestamp >= datetime('now', '-1 hour')
            """).fetchone()[0] or 0

            # 过去 24 小时平均错误率
            baseline = conn.execute("""
                SELECT
                    SUM(CASE WHEN status = 'error' THEN 1 ELSE 0 END) * 1.0 / COUNT(*) as error_rate
                FROM llm_calls
                WHERE timestamp >= datetime('now', '-24 hours')
                AND timestamp < datetime('now', '-1 hour')
            """).fetchone()[0] or 0

        anomalies = []
        if baseline > 0 and current / baseline > 3.0:
            anomalies.append({
                "type": "error_rate_spike",
                "severity": "critical",
                "current_rate": round(current * 100, 2),
                "baseline_rate": round(baseline * 100, 2),
                "message": f"当前错误率 {current*100:.1f}% 是基线 {baseline*100:.1f}% 的 {current/baseline:.1f} 倍"
            })

        return anomalies
```

```sql
WHERE DATE(timestamp) < CURRENT_DATE - INTERVAL '30 days'
AND NOT EXISTS (
    SELECT 1 FROM cost_records cr
    WHERE cr.record_date = DATE(call_logs.timestamp)
    AND cr.user_id = call_logs.user_id
    AND cr.model = call_logs.model
)
GROUP BY DATE(timestamp), user_id, model
ON CONFLICT (record_date, user_id, model) DO UPDATE SET
    total_calls = EXCLUDED.total_calls,
    total_tokens = EXCLUDED.total_tokens,
    total_cost_usd = EXCLUDED.total_cost_usd,
    avg_latency_ms = EXCLUDED.avg_latency_ms,
    error_count = EXCLUDED.error_count,
    updated_at = NOW();

-- 2. 删除已聚合的原始数据
DELETE FROM call_logs WHERE DATE(timestamp) < CURRENT_DATE - INTERVAL '30 days';
```
