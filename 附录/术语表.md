# 术语表

> **本附录解释培养计划中涉及的关键术语**

---

## A

### Agent
智能体。能够感知环境、做出决策、执行动作的 AI 系统。区别于单纯的 LLM，Agent 具有自主性和工具调用能力。

### Agent Loop
Agent 循环。Agent 处理任务的基本流程：Thought（思考）→ Action（行动）→ Observation（观察）→ 重复。

### API (Application Programming Interface)
应用程序接口。不同软件之间互相调用的方式。在本培养计划中，主要指 LLM API（OpenAI、Claude 等）。

### AutoGen
微软开源的多 Agent 框架。本培养计划项目 6 使用的框架。

---

## B

### BM25
经典的关键词检索算法。常和向量检索组合使用（混合检索）。

### Buffer Memory
缓冲记忆。短期记忆的一种，保留最近 N 轮对话。

---

## C

### Chain-of-Thought (CoT)
思维链。通过让 LLM 展示推理过程来提升复杂任务的准确性。

### Chroma
开源向量数据库。本培养计划项目 3 使用的向量数据库。

### Chunk
文本块。文档切分后的最小单位。

### CI/CD
持续集成/持续部署。自动化的代码测试和部署流程。

### Context Window
上下文窗口。LLM 一次能处理的最大 Token 数。

### Cosine Similarity
余弦相似度。衡量两个向量相似程度的指标，范围 -1 到 1。

### CrewAI
多 Agent 框架。本培养计划项目 6 的备选框架。

---

## D

### DAN (Do Anything Now)
一种 Prompt 注入攻击方式，尝试让 LLM 扮演无限制的角色。

### Docker
容器化平台。本培养计划项目 7 的核心技术。

### Docker Compose
Docker 的多容器编排工具。

### Dockerfile
Docker 镜像的构建脚本。

---

## E

### Embedding
嵌入。把文本转换为向量的过程。

### E2B
云端代码沙箱服务。本培养计划项目 5 使用。

### End-to-End (E2E)
端到端。从用户输入到最终输出的完整流程。

---

## F

### FastAPI
Python 的 Web 框架。本培养计划项目 10 推荐的 API 框架。

### Few-Shot Learning
少样本学习。在 Prompt 中给出几个示例，让 LLM 学会完成任务。

### Fine-tuning
微调。在预训练模型基础上，用特定数据继续训练。

### Function Calling
函数调用。LLM 调用外部函数/工具的能力。

---

## G

### GIL (Global Interpreter Lock)
Python 的全局解释器锁。影响 Python 多线程性能。

### GPT (Generative Pre-trained Transformer)
OpenAI 的生成式预训练 Transformer 模型系列。

### Grafana
可视化监控工具。本培养计划项目 7-8 使用。

---

## H

### Hallucination
幻觉。LLM 编造看似合理但实际错误的内容。

### HNSW (Hierarchical Navigating Small World)
一种向量索引算法，检索速度快。

### Human-in-the-Loop
人在回路。关键决策点需要人工确认的流程。

---

## I

### Inference
推理。LLM 接收输入并生成输出的过程。

### Instruction Tuning
指令微调。让 LLM 学会遵循人类指令的训练。

### Integration Test
集成测试。测试多个模块协同工作的测试。

---

## J

### Jailbreak
越狱。通过特殊 Prompt 让 LLM 绕过安全限制。

### JSON Schema
JSON 数据结构的定义规范。用于定义函数/工具的参数。

---

## K

### K8s (Kubernetes)
容器编排平台。本培养计划项目 7 挑战版使用。

---

## L

### LangChain
流行的 LLM 应用开发框架。

### LangGraph
LangChain 推出的 Agent 状态机框架。

### LangSmith
LangChain 推出的可观测性平台。

### LLM (Large Language Model)
大语言模型。GPT、Claude、Llama 等。

### LlamaIndex
RAG 应用开发框架。本培养计划项目 3 使用的框架。

---

## M

### MLOps
机器学习运维。把 DevOps 理念应用到 ML 系统。

### MCP (Model Context Protocol)
Anthropic 提出的 LLM 工具调用协议。

### Memory
记忆。Agent 存储和检索历史信息的能力。

### Multi-Agent
多 Agent。多个 Agent 协作完成任务。

---

## N

### NER (Named Entity Recognition)
命名实体识别。从文本中识别出人名、地名、机构名等。

### Nginx
高性能 Web 服务器，常用于反向代理。

---

## O

### Observability
可观测性。通过日志、指标、链路追踪理解系统行为。

### OCR (Optical Character Recognition)
光学字符识别。从图片中提取文字。

### OpenTelemetry
开源的可观测性标准。

---

## P

### P95 / P99
第 95/99 百分位的延迟。用于衡量系统性能。

### PII (Personally Identifiable Information)
个人可识别信息。如姓名、身份证号、手机号。

### Playground
在线调试工具。OpenAI、Anthropic 等都提供。

### Prompt
提示词。输入给 LLM 的文本。

### Prompt Engineering
Prompt 工程。设计有效的 Prompt 的技术和方法。

### Prompt Injection
Prompt 注入。攻击者通过特殊输入劫持 LLM 行为的攻击。

### Pytest
Python 的测试框架。

---

## Q

### QPS (Queries Per Second)
每秒查询数。衡量系统吞吐量的指标。

---

## R

### RAG (Retrieval-Augmented Generation)
检索增强生成。通过检索外部知识增强 LLM 回答的技术。

### ReAct
Reasoning + Acting。Agent 的一种推理-行动模式。

### Recall@K
检索评估指标。前 K 个结果中包含正确答案的比例。

### Red Teaming
红队测试。用攻击者思维测试系统安全性。

### Reinforcement Learning
强化学习。通过奖励信号训练模型。

### Rerank
重排序。对检索结果重新排序，提升相关性。

### REST API
基于 HTTP 的 API 设计风格。

### ROUGE
文本生成评估指标。用于评估摘要质量。

---

## S

### Sandbox
沙箱。隔离的执行环境，用于安全运行不可信代码。

### SLO (Service Level Objective)
服务水平目标。如可用性 99.9%。

### SLA (Service Level Agreement)
服务水平协议。服务提供方和用户之间的约定。

### Streamlit
Python 的 Web 应用框架。快速构建数据应用。

### Summary Memory
摘要记忆。把历史对话压缩为摘要的记忆策略。

### System Prompt
系统提示词。设定 LLM 角色和行为的基础 Prompt。

---

## T

### Temperature
温度参数。控制 LLM 输出的随机性，0-1 之间。

### Token
LLM 处理文本的最小单位。1 个 Token ≈ 0.75 个英文单词 ≈ 1.5 个中文字符。

### Token Limit
Token 限制。LLM 一次能处理的最大 Token 数。

### Tool Use
工具使用。LLM 调用外部工具的能力。

### Top-K / Top-P
采样参数。控制 LLM 从多少个候选 Token 中选择。

### Tracing
链路追踪。记录请求在系统中的完整调用链。

### TTL (Time To Live)
生存时间。数据在缓存中的有效期限。

---

## U

### UI (User Interface)
用户界面。

### Unit Test
单元测试。测试单个函数/类的测试。

---

## V

### Vector Database
向量数据库。存储和检索向量的专用数据库，如 Chroma、Milvus、Pinecone。

### Vector Search
向量检索。基于相似度在向量空间中查找。

### vLLM
高性能 LLM 推理引擎。

---

## W

### Webhook
HTTP 回调。事件发生时向指定 URL 发送通知。

### Weaviate
开源向量数据库。

### Workflow
工作流。任务的执行流程。

---

## Z

### Zero-Shot Learning
零样本学习。不给示例，直接让 LLM 完成任务。

### 向量 (Vector)
在高维空间中表示数据点的数学表示。文本通过 Embedding 转换为向量。

### 智能体 (Agent)
见 Agent。
