## 🎯 培养总览

### 设计理念

1. **梯度递进**：从"用框架"到"懂原理"，从"Demo"到"生产"
2. **工程优先**：80% 关注部署、监控、性能、安全，20% 关注算法
3. **可衡量交付**：每个项目都有明确的"基础版"和"挑战版"标准
4. **真实场景**：所有项目都基于真实业务需求设计

### 培养路径

```
LLM 基础 → 工具调用 → RAG → 记忆系统 → 代码执行 → 多 Agent
   ↓
容器化部署 → 监控告警 → 安全评估 → 综合实战
```

### 难度曲线

| Phase | 周次 | 核心能力 | 难度 | 项目数 |
|-------|------|----------|------|--------|
| Phase 1 | 1-4 | LLM 基础、工具调用、RAG | ⭐⭐⭐ | 3 |
| Phase 2 | 5-8 | 状态管理、代码执行、多 Agent | ⭐⭐⭐⭐ | 3 |
| Phase 3 | 9-12 | 部署、监控、安全 | ⭐⭐⭐⭐⭐ | 3 |

## 🎓 能力模型与考核标准

### 三级能力模型

#### 🟢 L1 - 入门级（完成 Phase 1）
**能力描述**：
- 能调用 LLM API，理解基本概念（Token、Prompt、Context）
- 能用 LangChain/LlamaIndex 框架构建简单 Agent
- 理解 RAG 基本原理

**考核方式**：
- 提交 3 个项目代码 + 演示视频
- 通过基础问答测试（10 个问题）

#### 🟡 L2 - 工程师级（完成 Phase 2）
**能力描述**：
- 能独立设计 Agent 架构（单 Agent + 多 Agent）
- 理解状态管理、记忆系统、代码沙箱
- 能排查常见问题（幻觉、循环、工具调用失败）

**考核方式**：
- 提交 3 个项目代码 + 技术博客
- 完成 1 个 Code Review 任务（审查他人代码）
- 通过进阶问答测试（20 个问题）

#### 🔴 L3 - 生产级（完成 Phase 4）
**能力描述**：
- 能从 0 到 1 设计并部署生产级 Agent 系统
- 掌握容器化、监控、安全、性能优化
- 能独立解决生产环境的复杂问题

**考核方式**：
- 提交项目 10 完整交付物
- 现场答辩（30 分钟，展示 + Q&A）
- 同行评审 + 专家评审

### 项目 1：LLM 应用初体验 —— 文档问答 Web 应用

**📌 任务描述**

构建一个 Web 应用，用户可以上传 PDF 文档，然后基于文档内容进行问答。

**🛠️ 技术栈**

- **前端**：Streamlit（最快出界面）
- **后端**：Python 3.10+
- **LLM API**：OpenAI GPT-4 / Claude / 国产大模型（选一个）
- **PDF 处理**：PyPDF2 或 pdfplumber

**📚 学习重点**

1. LLM API 调用方式（`chat.completions.create`）
2. Prompt 设计基础（System Message、User Message、Assistant Message）
3. Token 概念与成本控制
4. Streamlit 交互式界面开发
5. 错误处理（API 限流、网络错误）

**✅ 基础版任务（必做）**

- [ ] 调用 LLM API 完成一次简单对话
- [ ] 实现 PDF 上传功能
- [ ] 提取 PDF 文本内容
- [ ] 设计 Prompt 模板，将文档内容作为 Context
- [ ] 用 Streamlit 构建对话界面
- [ ] 实现多轮对话（保持对话历史）

**🚀 挑战版任务（选做 2 个即可）**

- [ ] 支持多个 PDF 同时上传
- [ ] 回答时引用原文位置（页码、段落）
- [ ] 支持流式输出（打字机效果）
- [ ] 加入"重新生成"和"复制"按钮
- [ ] 对话历史导出（Markdown / PDF）

**📊 评估标准**

| 等级 | 标准 |
|------|------|
| 及格 | 能上传 PDF 并回答问题，基础功能可用 |
| 良好 | 支持多文档、引用原文、界面美观 |
| 优秀 | 流式输出、Token 监控、错误处理完善 |

**💰 预估成本**

- OpenAI GPT-4o-mini：约 $0.15/1M tokens
- 完整开发测试：约 $1-3

**📦 交付物**

1. 完整可运行代码（GitHub Repo）
2. README.md（包含启动方式、配置说明）
3. 演示视频（5 分钟以内）

1. Function Calling / Tool Use 原理

5. 错误处理（工具调用失败怎么办）

**✅ 基础版任务（必做）**

- [ ] 实现命令行交互界面（输入 → 输出）
- [ ] 处理工具调用失败的情况
- [ ] 记录 Agent 的执行过程（Thought/Action/Observation）

**🚀 挑战版任务（选做 2 个即可）**

- [ ] 加入"记忆"功能，记住用户偏好
- [ ] 支持工具组合调用（"北京和上海的天气对比"）

- [ ] 工具调用性能优化（并行调用独立工具）

| 等级 | 标准 |

| 及格 | 能调用 3 个工具完成基础任务 |
| 良好 | 工具组合调用、错误处理完善 |

### 项目 3：RAG 系统 —— 技术文档助手

**📌 任务描述**

为公司技术文档库（或任意文档集合）构建一个 RAG（检索增强生成）问答系统。

**🛠️ 技术栈**

- **框架**：LlamaIndex（推荐）或 LangChain
- **向量数据库**：Chroma（本地）/ Pinecone / Qdrant
- **Embedding**：OpenAI text-embedding-3-small（便宜）或其他开源模型
- **LLM**：GPT-4o-mini / Claude Haiku（成本考虑）

**📚 学习重点**

1. RAG 全流程：文档加载 → 切分 → Embedding → 检索 → 生成
2. 文档切分策略（按段落、按 Token、按语义）
3. 向量检索原理（Cosine Similarity）
4. Context 注入 Prompt 的技巧
5. RAG 评估方法（答案是否基于原文）

**✅ 基础版任务（必做）**

- [ ] 准备 10+ 份技术文档（Markdown/PDF/HTML）
- [ ] 实现文档加载与切分
- [ ] 构建向量索引（Chroma 本地存储）
- [ ] 实现相似度检索（Top-K）
- [ ] 设计 Prompt，将检索结果作为 Context
- [ ] 构建问答界面（命令行或 Streamlit）

**🚀 挑战版任务（选做 2 个即可）**

- [ ] 加入 Rerank 模型（用 Cohere Rerank 或 bge-reranker）
- [ ] 支持混合检索（向量 + 关键词 BM25）
- [ ] 答案引用原文位置（文件名、段落号）
- [ ] 多文档联合问答（"综合 A 和 B 两个文档"）
- [ ] 检索质量评估（构建测试集，评估 Recall@K）

**📊 评估标准**

| 等级 | 标准 |
|------|------|
| 及格 | 能基于文档回答问题，能引用原文 |
| 良好 | 检索准确率高、答案质量好 |
| 优秀 | Rerank、混合检索、评估体系完善 |

**💰 预估成本**

- Embedding：约 $0.02/1M tokens（text-embedding-3-small）
- LLM 调用：取决于问答次数，开发测试约 $2-5

**📦 交付物**

1. 完整代码 + 文档数据
2. 检索效果演示（至少 5 个典型问题）
3. 简短技术博客（讲清楚 RAG 原理和你的优化点）

### 项目 4：有记忆的对话 Agent

**📌 任务描述**

- [ ] 记忆用户基本信息（姓名、职业、偏好）

**🚀 挑战版任务（选做 2 个即可）**

### 项目 5：代码执行 Agent —— 数据分析助手

**📌 任务描述**

构建一个能自动写 Python 代码并执行的数据分析助手。用户上传 CSV，Agent 自动分析数据并生成图表。

**🛠️ 技术栈**

- **沙箱**：E2B（推荐，开箱即用）或 Pyodide（本地）
- **Agent 框架**：LangChain
- **可视化**：Matplotlib / Plotly
- **数据处理**：Pandas

**📚 学习重点**

1. 沙箱隔离的重要性（不能直接 `exec`）
2. 代码生成与语法检查
3. 执行错误捕获与自我修复
4. 资源限制（CPU、内存、时间）
5. 代码安全（禁止访问文件系统、限制网络）

**✅ 基础版任务（必做）**

- [ ] 集成 E2B 沙箱（或类似服务）
- [ ] Agent 根据用户问题生成 Python 代码
- [ ] 在沙箱中执行代码
- [ ] 捕获执行错误，让 Agent 自我修复
- [ ] 返回代码 + 执行结果 + 图表

**🚀 挑战版任务（选做 2 个即可）**

- [ ] 支持多轮分析（基于上一轮结果继续分析）
- [ ] 单元测试自动生成（Agent 写完代码后自动生成测试）
- [ ] 代码质量评估（复杂度、可读性）
- [ ] 支持多种数据格式（Excel、JSON、Parquet）
- [ ] 沙箱资源监控（每次执行的 CPU/内存使用）

**📊 评估标准**

| 等级 | 标准 |
|------|------|
| 及格 | 能生成代码、执行、返回结果 |
| 良好 | 错误自我修复、图表美观 |
| 优秀 | 单元测试、资源监控、代码质量评估 |

**💰 预估成本**

- E2B 沙箱：免费额度 + 付费版（$0.10/小时起）
- LLM 调用：约 $5-10（代码生成 Token 消耗大）

**📦 交付物**

1. 完整代码 + 沙箱配置
2. 演示视频（展示完整的数据分析流程）
3. 安全测试报告（证明沙箱隔离有效）

### 项目 7：容器化与部署

**📌 任务描述**

将项目 6 的多 Agent 系统打包成 Docker 镜像，部署到云服务器，并配置 CI/CD。

**🛠️ 技术栈**

- **容器化**：Docker、Docker Compose
- **编排**（挑战版）：Kubernetes / Docker Swarm
- **CI/CD**：GitHub Actions / GitLab CI

## ❓ 常见问题 FAQ

### Q1: 没有 LLM 基础，能直接开始吗？

**A**: 建议先花 1-2 周学习 Python 基础和 LLM 基本概念：
- Python 异步编程（asyncio）
- HTTP 请求（requests）
- JSON 数据处理
- LLM 概念（Token、Prompt、Context Window）

### Q2: 没有 GPU，能跑开源模型吗？

**A**: 本培养计划主要使用 API（OpenAI/Claude/国产大模型），不需要本地 GPU。如果想用开源模型：
- 小模型（7B）：可以用 M2 Mac 或 CPU 运行
- 大模型（70B+）：需要云 GPU（AutoDL / 恒源云）

### Q3: 预算有限，怎么降低 API 成本？

**A**:
- 使用便宜的模型（GPT-4o-mini、Claude Haiku、Qwen-Turbo）
- 加入缓存（相同问题直接返回）
- 控制 Context 长度（不要无脑塞历史对话）
- 使用国产模型（DeepSeek、Qwen 比 OpenAI 便宜）

### Q4: 每周 15-20 小时够吗？

**A**:
- Phase 1-2：基本够用，可能需要加班
- Phase 3：建议 20-25 小时（部署、监控需要时间踩坑）
- Phase 4：建议 25-30 小时（综合项目工作量大）

### Q5: 一个人做还是组队？

**A**:
- 建议**前 3 个项目**一个人做（打基础）
- **项目 4-6** 可以 2 人组队（互相 Code Review）
- **项目 7-10** 建议 3 人组队（分工明确：一个人负责部署、一个人负责监控、一个人负责安全）

### Q6: 完成所有项目后能达到什么水平？

**A**: 能独立完成以下工作：
- 设计并实现中等复杂度的 Agent 系统
- 部署到云服务器并保证稳定性
- 排查生产环境的常见问题
- 跟踪最新的技术进展并评估是否引入

### Q7: 怎么找工作？需要刷算法题吗？

**A**:
- **Agent 研发岗**：更看重项目经验，建议把项目 6、8、10 作为面试重点
- **算法题**：LeetCode 中等难度 100 题左右即可
- **加分项**：技术博客（讲清楚项目原理和踩坑经验）

### Q8: 这个计划能根据个人情况调整吗？

**A**: 当然可以。建议：
- **已掌握 LLM 基础**：可以跳过项目 1-2，直接从项目 3 开始
- **已掌握部署**：可以跳过项目 7 的基础版，直接做挑战版
- **有特定方向**（如金融、医疗）：可以将项目 10 选题聚焦到该领域
