## 一、为什么要从「文档问答」开始？

老板，讲这个项目前，**先帮学员建立正确的预期**——这不是一个"玩具项目"，而是一个**真正能用的工具**。

开场可以这样讲：

> "你写论文、读合同、看产品手册，是不是经常要翻来翻去找一段具体内容？现在我们用 1-2 周时间，做一个工具：上传任何文档，直接问它问题。这玩意儿市面上卖几百块一年，我们自己做一个。"

**这个项目有 3 个不可替代的价值**：

1. **跑通完整链路**——文件上传、文本解析、Prompt 构造、API 调用、结果返回，这套流程 90% 的 LLM 应用都在用
2. **建立成本意识**——第一次看到「一次提问要 4 分钱」时的震惊，是最好的成本课
3. **直面幻觉问题**——文档里没写的内容，模型会编。这个坑早踩早免疫

## 二、3 个核心概念，必须讲透

### 概念 1：Prompt ≠ 「跟 AI 说话」

很多学员的误区：把 Prompt 当成跟 ChatGPT 聊天时的随口一问。

**讲师要强调**：

> Prompt 是**给模型的程序**。System Message 定义了模型的"人设"和"行为规则"，User Message 是输入。**Prompt 工程 = 给模型写说明书 + 写需求文档**。

可以举这个反例：

```python
# 烂 Prompt
"看看这份文档说了啥"

# 好 Prompt
SYSTEM = """你是文档问答助手。规则：
1. 只回答文档里有的内容
2. 引用具体段落
3. 不超过 200 字
"""

USER = """文档：[文档内容]
问题：这份文档的核心观点是什么？
"""
```

让学员**亲眼看到**：同样的问题，不同 Prompt，回答质量天差地别。

### 概念 2：Token ≠ 字符

英文 1 Token ≈ 4 字符 ≈ 0.75 单词
中文 1 Token ≈ 1-2 字符

**为什么要强调**：
- 模型的"上下文窗口"是按 **Token** 算的（GPT-4o 是 128K Token）
- **价格是按 Token 算的**（gpt-4o 输入 $2.5/M Token）
- **学员第一次用 GPT-4 时，看到账单就知道心疼**——这是最好的成本教育

**演示环节**：用 `tiktoken` 库让学员亲眼数一段文本的 Token 数。

```python
import tiktoken
enc = tiktoken.encoding_for_model("gpt-4o")
text = "你好，世界！Hello, world!"
print(f"字符数: {len(text)}, Token数: {len(enc.encode(text))}")
```

输出：`字符数: 14, Token数: 7` —— 中英文混合下的实际消耗。

### 概念 3：Context 窗口是"快闪记忆"

**一个反直觉的事实**：

> 即使模型支持 128K Context，把 128K 内容全塞进去，**后段内容容易被忽略**。

这叫 "Lost in the Middle" 现象。研究表明：

```
位置:    [开头]  [1/4]  [2/4]  [3/4]  [结尾]
召回率:   85%    78%    60%    75%    88%
```

**讲师要点**：
- 不是 Token 越多越好
- 文档切分 + 精准检索，比无脑塞全文更有效
- **这就是项目 3（RAG）要解决的问题**——让项目 1 的学员带着这个问题进入项目 3

## 三、代码讲解节奏（建议 60/30/10 分配）

### 60% 时间：手把手写最小闭环

**别让学员看完整代码再开始**。建议节奏：

```
讲师讲 1 行 → 学员跟 1 行 → 跑一下 → 看效果 → 继续
```

**第一个 demo**（15 分钟）：终端直接跟 GPT 聊天

```python
from openai import OpenAI
client = OpenAI()

resp = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "你好"}]
)
print(resp.choices[0].message.content)
```

**第二个 demo**（20 分钟）：读取本地 txt，问问题

```python
with open("doc.txt") as f:
    doc = f.read()

resp = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": f"基于以下文档回答：\n{doc}"},
        {"role": "user", "content": "文档讲了什么？"}
    ]
)
print(resp.choices[0].message.content)
```

**第三个 demo**（25 分钟）：升级到 Streamlit + PDF

这里要**慢一点**，把 Streamlit 的 `st.file_uploader`、`st.chat_input`、`st.write_stream` 这几个函数都过一遍。

### 30% 时间：讲 3 个工程实践

这部分**比写代码更重要**，是初级工程师成长为中级工程师的分水岭：

#### 实践 1：API Key 管理

```python
# ❌ 烂做法
client = OpenAI(api_key="sk-xxxxxx")  # 提交到 GitHub 直接被刷爆

# ✅ 正确做法
import os
from dotenv import load_dotenv
load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))
```

必须配套 `.gitignore`：
```
.env
__pycache__/
*.pyc
```

**讲师金句**：「你写代码的时间占 30%，但泄漏一个 Key 损失 3000 块，值不值这个习惯？」

#### 实践 2：Token 计数

让学员在每次调用前后打印 Token 数：

```python
import tiktoken

def count_tokens(text, model="gpt-4o-mini"):
    return len(tiktoken.encoding_for_model(model).encode(text))

# 调用前
prompt = f"系统提示...\n\n问题：{question}"
print(f"输入 Token: {count_tokens(prompt)}")

# 调用后
print(f"输出 Token: {resp.usage.completion_tokens}")
print(f"总消耗: {resp.usage.total_tokens}")
```

**这 5 行代码的培养价值，比调 50 个 API 调试参数都高**。学员开始有"成本感"。

#### 实践 3：错误处理

直接演示"不处理错误会怎样"——断网、调快 API 速率、上传损坏文件。

然后教最实用的 try/except + 重试模板（架构书里有完整代码）。

## 四、必讲的 5 个踩坑点

### 坑 1：Prompt 塞太多规则

学员常犯：System Prompt 写 500 字规则。
- **问题**：规则之间可能矛盾，模型困惑
- **对策**：规则**不超过 5 条**，每条**不超过 1 句话**

### 坑 2：用 `print` 打印长文档

```python
print(document_text)  # 1000 页的 PDF 打印到哪？
```

**对策**：
```python
print(f"文档长度: {len(doc)} 字符")
print(f"前 500 字: {doc[:500]}")
print(f"Chunk 数量: {len(chunks)}")
```

### 坑 3：忘记 `stream=True`

学员的体验：上传文档后，等 10 秒才看到答案。
**对策**：用流式输出 + Streamlit 的 `st.write_stream`，逐字展示。

### 坑 4：温度参数拍脑袋

学员常困惑：`temperature` 该设多少？

| 场景 | 推荐值 | 理由 |
|------|--------|------|
| 文档问答 | **0.1-0.3** | 准确 > 创造性 |
| 创意写作 | 0.7-0.9 | 多样性 > 准确 |
| 代码生成 | 0.0-0.2 | 要确定性输出 |

### 坑 5：没测试就交付

学员最常被批评的：写完代码直接交，没测试。

**强制要求**：
- 至少 10 个测试问题
- 8 个以上回答正确才算通过
- 把测试问题写进 README

## 六、Q&A 预案

**Q1：为什么不用 LangChain？**
A：项目 1 故意不用框架。直接用 OpenAI SDK 让学员理解**底层原理**——Prompt 是什么、API 怎么调。LangChain 是封装，**先会走路再学开车**。项目 2 开始引入 LangChain。

**Q2：gpt-4o-mini 够用吗？**
A：日常问答够用。复杂推理（数学、多步逻辑）才需要 gpt-4o。建议**先 mini 跑通，按需升级**。

**Q3：怎么防止模型编造？**
A：三层防御——①Prompt 加规则 ②强制引用原文 ③关键信息用 temperature=0。

**Q4：用户上传敏感文档怎么办？**
A：本期不处理，**项目 9（安全）**专门讲。提示学员：生产环境必须加隐私保护 + 用户协议。

**Q5：能换成 Claude 吗？**
A：能。OpenAI SDK 和 Anthropic SDK 接口类似，改 3 行代码就行。**演示一下**，让学员看到"换模型没那么难"。
